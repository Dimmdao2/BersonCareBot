Назначение слоя

Интеграции связывают систему с внешними сервисами.

Примеры:
	•	Telegram
	•	Rubitime
	•	SMS
	•	Email

Основная задача

Интеграции выполняют две функции:
	1.	принимают события извне
	2.	отправляют сообщения наружу

Входящие события

Интеграция:
	1.	принимает webhook
	2.	проверяет токены
	3.	валидирует данные
	4.	преобразует их в единый формат события

После этого событие передаётся в eventGateway.

webhook
  ↓
integration
  ↓
IncomingEvent
  ↓
eventGateway

Исходящие сообщения

Интеграция получает универсальное сообщение (intent) и:
	1.	формирует текст
	2.	преобразует структуру в формат API
	3.	отправляет сообщение

intent
  ↓
integration
  ↓
API сервиса

Что интеграции НЕ делают
	•	не принимают бизнес-решения
	•	не читают базу данных
	•	не выполняют сценарии
	•	не управляют ретраями
	•	не знают оркестратор

Главное правило

Интеграции — это тонкий слой адаптеров.
