Queue — инфраструктурный слой управления очередями задач.

Очередь используется для передачи задач между доменом и worker.

Основные задачи

Queue обеспечивает:
•	хранение задач
•	передачу задач worker
•	управление повторными попытками
•	планирование отложенных задач

Типы задач
Основной тип задач:delivery job

Пример задачи:
send message
retry policy
delay
attempt

Роль очереди

Очередь выполняет только инфраструктурную функцию.
Она:
•	хранит задачи
•	выдаёт задачи worker

Что очередь НЕ делает

Queue:
•	не принимает решений
•	не знает бизнес-логики
•	не знает сценарии
•	не знает пользователей

Она хранит только задачи выполнения.

Источник задач

Задачи создаются доменом.

Worker получает задачи из очереди.
