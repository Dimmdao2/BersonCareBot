# DB Layer

Слой `src/infra/db` отвечает только за low-level работу с БД, репозитории и запуск миграций.

## Разделение миграций

- **Core migrations** (общие таблицы, не привязанные к конкретной интеграции):
  - `src/infra/db/migrations/core/*.sql`
- **Integration migrations** (таблицы конкретной интеграции):
  - `src/integrations/<name>/db/migrations/*.sql`

Мигратор `src/infra/db/migrate.ts` применяет миграции автоматически:

1. Сначала `core`.
2. Затем миграции всех интеграций из `src/integrations/*/db/migrations` в лексикографическом порядке по имени интеграции.
3. Внутри каждой папки — по имени файла (лексикографически).

## Правило добавления новой интеграции

Если интеграции нужны таблицы в БД:

1. Добавить SQL-файлы только в `src/integrations/<name>/db/migrations`.
2. При необходимости добавить `src/integrations/<name>/db/schema.md`.
3. Не менять `src/infra/db/**` для простого добавления интеграции.

## Границы

`src/infra/db/**`:

- не содержит бизнес-логики;
- не знает сценарии orchestrator/domain;
- не отправляет сообщения напрямую.
