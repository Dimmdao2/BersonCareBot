Н# Runtime / Worker subsystem

Этот документ описывает подсистему выполнения фоновых задач системы.

Подсистема runtime отвечает за выполнение действий, которые не должны выполняться во время HTTP-запроса или обработки webhook.

К таким действиям относятся:

- отправка сообщений
- повторные попытки доставки
- отложенные действия
- задачи по расписанию
- длительные операции

Runtime не содержит бизнес-логики.  
Он выполняет задачи, созданные доменом.

---

# Общая схема работы

Общий поток выполнения задачи:

domain
↓
создание job
↓
queue / storage
↓
worker
↓
jobExecutor
↓
domain.executeAction
↓
dispatcher
↓
integration
↓
external API

---

# Роль runtime

Runtime отвечает только за:

- когда выполнять задачу
- сколько раз повторять задачу
- когда остановить повторы

Runtime **не решает**:

- какую задачу создать
- какой сценарий выполнять
- что отправлять пользователю

Все решения принимает **домен и оркестратор**.

# worker.ts

## Назначение

Главный процесс выполнения фоновых задач.

Worker постоянно проверяет очередь задач и запускает их выполнение.

## Основные функции

Worker:

- читает задачи из очереди
- передает задачу в jobExecutor
- получает результат выполнения
- применяет retryPolicy
- планирует следующую попытку при необходимости

## Основной цикл worker

loop
↓
получить job из очереди
↓
jobExecutor.execute(job)
↓
получить результат
↓
retryPolicy
↓
решение:
завершить
повторить
запланировать

Worker не знает бизнес-логики задачи.

Он работает только с инфраструктурными правилами выполнения.

---

# jobExecutor.ts

## Назначение

Исполнитель конкретной задачи.

jobExecutor получает задачу и выполняет её через домен.

## Основная задача

jobExecutor выполняет один job.

job
↓
domain.executeAction
↓
integration
↓
result

jobExecutor не принимает решений о повторных попытках.

Он только выполняет действие и возвращает результат.

## Пример задачи

send_message
channel: telegram
template: appointment_reminder
recipientId: 123
attempt: 1

jobExecutor вызывает домен:

domain.executeAction(job)

После выполнения возвращает:

success
failed
temporary_error

---

# retryPolicy.ts

## Назначение

Определяет правила повторных попыток выполнения задач.

retryPolicy отвечает на вопрос:

- можно ли повторить задачу
- когда выполнить следующую попытку

## Источник настроек

Настройки retry берутся из:

config/appSettings.json

Пример конфигурации:

delivery:
maxAttempts: 3
retryDelaySeconds: 60

## Основные функции

retryPolicy:

- проверяет количество попыток
- определяет время следующей попытки
- решает нужно ли остановить выполнение

## Пример

attempt = 1
maxAttempts = 3

→ повтор разрешён

nextRetry = now + 60 seconds

Если:

attempt = 3
maxAttempts = 3

→ задача завершается.

# scheduler

## Назначение

Scheduler отвечает за задачи, которые должны выполниться в определённое время.

Например:

- напоминание через 1 час
- сообщение завтра
- ежедневные задачи

## Работа scheduler

Scheduler проверяет задачи со статусом "scheduled".

Когда наступает время выполнения:

scheduled job
↓
queue
↓
worker

---

# Dispatcher

Dispatcher используется worker при отправке сообщений.

Он выбирает нужную интеграцию для отправки.

intent
↓
dispatcher
↓
integration

Dispatcher не содержит бизнес-логики.

Он только определяет нужный канал.

---

# Типы задач (jobs)

Система поддерживает несколько типов задач.

Примеры:

send_message
send_email
send_sms
calendar_sync
cleanup

Каждый тип задачи выполняется через:

jobExecutor
↓
domain action

---

# Главное правило runtime

Runtime выполняет задачи.

Он **никогда не принимает бизнес-решения**.

Решения принимает:

orchestrator
domain

Runtime обеспечивает только механизм выполнения.

---

# Разделение ответственности

orchestrator
решает что делать

domain
знает как выполнить действие

runtime
выполняет действие в нужное время

integrations
общаются с внешними сервисами

---

# Причина существования worker

Worker нужен чтобы:

- не блокировать HTTP обработку
- выполнять долгие операции
- выполнять отложенные действия
- безопасно повторять ошибки

Без worker система не сможет масштабироваться и безопасно выполнять задачи.
