Назначение слоя

Слой запуска приложения.

Здесь собирается всё приложение и соединяются все части системы.

Этот слой:
	•	запускает HTTP-сервер
	•	регистрирует маршруты
	•	создаёт зависимости
	•	соединяет интеграции, домен и инфраструктуру

Что делает
	1.	Запускает сервер.
	2.	Регистрирует webhook-маршруты.
	3.	Подключает зависимости.
	4.	Передаёт события из интеграций в систему.

Что НЕ делает
	•	не содержит бизнес-логики
	•	не работает с базой данных напрямую
	•	не выполняет сценарии
	•	не формирует сообщения

Поток данных

HTTP запрос
   ↓
route handler
   ↓
integration

Зависимости

Можно использовать:

integrations
kernel
infra
config

Нельзя:

писать бизнес-логику
обращаться к БД напрямую
