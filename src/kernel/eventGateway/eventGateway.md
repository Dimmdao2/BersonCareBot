# eventGateway

## Назначение

`eventGateway` — технический модуль ядра, принимающий **уже нормализованные события** от `integrations/*` и выполняющий базовую техническую обработку перед передачей события в домен.

EventGateway — единая точка входа событий в систему.

Он принимает события из интеграций и подготавливает их для обработки доменом.

Основные задачи
	1.	проверка структуры события
	2.	дедупликация
	3.	rate limit
	4.	добавление системных метаданных

Что делает

Принимает:

IncomingEvent

Проверяет:
	•	корректность структуры
	•	повтор события
	•	лимиты

После проверки:

event → domain

Возможные результаты

accepted
accepted_noop
rejected
dropped

Что НЕ делает

EventGateway:
	•	не принимает бизнес-решения
	•	не вызывает оркестратор
	•	не читает базу данных
	•	не отправляет сообщения

Главное правило

Gateway только фильтрует и нормализует события.
