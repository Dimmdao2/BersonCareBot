## Цели ветки

1. **Чистка UI/UX** — убрать заглушки, мок-данные, дублирование из навигации
2. **Полноценная CMS + медиабиблиотека** — множественная загрузка, grid-view, drag-and-drop, модальный picker, мобильный capture
3. **Нативный модуль записи на приём** — заменить Rubitime iframe собственным UI с синхронизацией расписания через Rubitime API + Google Calendar
4. **UX-улучшения кабинета врача** — объединение списков, пагинация, визуализация статистики

## Основные решения

### Модели для этапов

| Этап | Модель | Почему |
|---|---|---|
| Декомпозиция фазы → атомарные задачи | GPT 5.3 / Sonnet 4.6 Opus High-Thinking | Глубокий анализ кода, генерация спеков с пониманием контекста |
| Реализация задач | Composer 1.5 (auto) | Массовая генерация кода по чётким спекам, оптимальная цена |
| Аудит / code review | GPT 5.3 Codex | Лучшее соотношение качество/цена для review; 5.4 только если 5.3 пропускает баги |
| Мелкие фиксы после аудита | Composer 1.5 (auto) | Быстрые правки по конкретным замечаниям |

### Порядок фаз

```
Фаза 0: Чистка UI/UX (1-2 дня)
  └─ Декомпозиция → Реализация → Аудит
Фаза 1: CMS + Медиабиблиотека (5-8 дней)
  └─ Декомпозиция → Реализация блоками → Аудит каждого блока
Фаза 2: Нативный модуль записи (8-12 дней)
  └─ Спек → Декомпозиция → Backend → Frontend → Интеграция → Аудит
Фаза 3: UX кабинета врача (3-5 дней)
  └─ Декомпозиция → Реализация → Аудит
Фаза 4: Рассылки (3-5 дней)
  └─ Декомпозиция → Реализация → Аудит
```
