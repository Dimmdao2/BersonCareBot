## Fallback Policy

> **ВАЖНО:** seed script должен завершаться с ошибкой (`process.exit(1)`), если любой из обязательных Rubitime ID отсутствует или равен пустой строке.  
> **Не допускается** молчаливый пропуск (`skip`) записей с отсутствующими ID — это скрывает ошибки конфигурации.

| Поле | Обязательно | Поведение при отсутствии |
|---|---|---|
| `rubitime_branch_id` | **Да** | FAIL — прервать seed с ошибкой |
| `rubitime_cooperator_id` | **Да** | FAIL — прервать seed с ошибкой |

## 8. Idempotency key для seed

Seed script использует upsert на основе:

- `booking_cities`: ключ `code`
- `booking_branches`: ключ `rubitime_branch_id`
- `booking_specialists`: ключ `(rubitime_cooperator_id, branch_id)`
- `booking_services`: ключ `(title, duration_minutes)` — если title/duration совпадают, update price/description
- `booking_branch_services`: ключ `(branch_id, service_id)`
