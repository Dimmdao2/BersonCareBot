## 5. Composer-Safe UI contract

### Doctor UI

Use only:

- `Label` from `@/components/ui/label`
- `Textarea` from `@/components/ui/textarea` for `goals` and `objectives`
- `Input` from `@/components/ui/input` for duration fields
- `Button` from `@/components/ui/button`

Required local structure for the stage-fields block:

```tsx
<div className="rounded-md border border-border/60 bg-muted/20 p-3">
  <div className="flex flex-col gap-3">
    ...
  </div>
</div>
```

Labels:

- `Цель этапа`
- `Задачи этапа`
- `Ожидаемый срок, дней`
- `Ожидаемый срок, текстом`

Help text class: `text-xs text-muted-foreground`.

Forbidden:

- raw `<button>` for actions;
- raw `<textarea>`/`<input>` instead of shadcn primitives;
- adding new markdown editor dependency;
- changing item add modal in A1.

### Patient UI

Use from `@/shared/ui/patientVisual`:

- `patientSectionSurfaceClass`
- `patientSectionTitleClass`
- `patientBodyTextClass`
- `patientMutedTextClass`

Do not import from `app/app/patient/home/*`.

Empty values:

- If all fields empty, render nothing.
- If only one field is set, render only that field.
- Do not show empty headings.

## 8. Rollback notes

- Migration is additive nullable. Rollback can ignore new columns in code first, then drop columns in a follow-up migration if needed.
- Do not make patient UI depend on field presence without null guards.

## 9. Definition of Done

- Fields exist on template and instance stages.
- Copy template->instance preserves values.
- Doctor can edit values.
- Patient sees non-empty values in stage header.
- Empty fields do not create empty UI.
- LOG updated using `LOG_TEMPLATE.md`.
