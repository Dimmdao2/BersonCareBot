## 2. Главная «Сегодня» — `/app/patient`

**Задача:** домашняя витрина дня; для авторизованного пользователя с персональным tier — персональные блоки; для гостя — ограниченная non-personal витрина.

**Блоки** задаются в БД (`patient_home_blocks` / `patient_home_block_items`), порядок и видимость конфигурируемы. Канонические коды блоков (`PatientHomeBlockCode`):

| Код | Смысл |
|-----|--------|
| `daily_warmup` | Карточка дневной разминки |
| `useful_post` | Полезный пост из CMS |
| `booking` | Запись на приём |
| `situations` | Ряд «ситуаций» (чипы из контента) |
| `progress` | Прогресс практики (цель дня, streak) |
| `next_reminder` | Ближайшее напоминание |
| `mood_checkin` | Чек-ин настроения |
| `sos` | Карточка SOS / важной темы |
| `plan` | Карточка активной программы лечения |
| `subscription_carousel` | Карусель подписок / офферов |
| `courses` | Ряд курсов |

Дополнительно: приветствие (`PatientHomeGreeting`, имя и время суток).

**Примечание:** в `navigation.ts` присутствует список `HomeBlockId` и `patientHomeBlocksCanonical` для legacy SSR-платформы — это **не** то же самое, что конфигурируемые блоки «Сегодня» из БД; актуальная сборка главной — `PatientHomeToday` + данные БД.

## 4. Смысловая склейка разделов

Цепочка продукта: **главная «Сегодня»** → **запись и визиты (cabinet / booking)** → **программы и курсы** → **дневник и напоминания** → **контент CMS** → **профиль и уведомления** → **поддержка и сообщения**. Часть URL сохранена только как редиректы на канонические маршруты.
