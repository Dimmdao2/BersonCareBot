# Продуктовые тексты и machine-reason (снимок для репозитория)

Каноничная полная версия с контекстом и статусом трека — в плане Cursor `webapp-first_phone_bind_5069b809` (локально в IDE: `~/.cursor/plans/…`).  
Здесь — **снимок таблицы** без привязки к машине разработчика; обновлять при изменении таксономии в коде (`PhoneLinkFailureReason`, `phoneLinkUserMessages.ts`). Операционные логи/метрики привязки и M2M emit: [`STAGE_05_OBSERVABILITY_TESTS_DOCS.md`](STAGE_05_OBSERVABILITY_TESTS_DOCS.md), контракт: [`apps/webapp/INTEGRATOR_CONTRACT.md`](../../apps/webapp/INTEGRATOR_CONTRACT.md).

| Ситуация | Machine reason | Что видит пользователь | CTA |
|----------|----------------|------------------------|-----|
| Временный сбой БД / TX | `db_transient_failure` (+ часто `phoneLinkIndeterminate`) | «Не удалось сохранить номер. Попробуйте позже…» | Повторить / поддержка |
| Нет binding (strict) | `no_channel_binding` | «Сначала откройте приложение из этого бота…» | Мини-приложение: inline «Открыть мини-приложение», если в контексте бота есть `facts.links.webappHomeUrl`; иначе только текст (в т.ч. «кнопка меню» в Telegram) |
| Нет строки `identities` для чата (гонка / рассинхрон) | `no_integrator_identity` | «Сессия бота не синхронизирована… /start…» | Мини-приложение / /start |
| Номер у другого аккаунта | `phone_owned_by_other_user` | «Номер уже привязан к другому аккаунту…» | Поддержка |
| Конфликт `integrator_user_id` | `integrator_id_mismatch` | «Не удалось сопоставить аккаунт с приложением…» | Поддержка |
| Неопределённо (нет метаданных от `writeDb`) | _(indeterminate)_ | Нейтральное «не удалось сохранить» | Повторить / поддержка |
| Успех | — | «Номер привязан» (сценарии) | — |

**Запрещено:** показывать текст конфликта номера при транзиентных ошибках БД.

**Целевой путь (одна TX):** не использовать отдельные reason вида «webapp недоступен» / `partial_apply_*` — они про legacy HTTP между процессами.
