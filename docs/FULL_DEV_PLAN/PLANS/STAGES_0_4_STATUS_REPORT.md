## Этап 3 — профиль и аутентификация

**Реализовано в коде:** inline-редактирование профиля, SMS OTP с лимитами, email challenge, API `/api/auth/phone/*`, `/api/auth/email/*`, channel-link (секреты в БД, завершение через integrator), тесты на ключевые сценарии, `hasMessengerBinding` для кабинета/уведомлений.

**Операционно:**

- На проде должны быть **применены миграции 016–018** до использования OTP/email/channel-link.
- При недоступности БД channel-link может вернуть URL без сохранения секрета — завершение привязки тогда не сработает (ожидаемое ограничение для degraded-среды).

**Исправление (безопасность/качество):** строгая проверка email на API старта email-аутентификации (см. резюме выше).

**Вывод:** этап 3 по функционалу и тестам — **выполнен**; остаётся дисциплина деплоя миграций.
