# Этап 12: Напоминания (Reminders)

> Приоритет: P2
> Зависимости: Этап 2 (дизайн-система), Этап 6 (дневники)
> Риск: низкий (backend уже частично реализован в integrator)

---

## Подэтап 12.1: UI — страница «Бот заботы»

**Задача:** страница настройки напоминаний для клиента.

**Файлы:**
- Новая страница: `/app/patient/reminders`
- Модуль: `apps/webapp/src/modules/reminders/`

**Действия:**
1. Кнопка «Бот заботы» на главной → `/app/patient/reminders`.
2. Страница содержит:
   - **Каналы напоминаний:** toggles для каждого привязанного канала (Telegram, Max, SMS, email, push — в будущем).
   - **Расписание:** выбор дней недели (чекбоксы), время (window start/end), интервал.
   - **О чём напоминать:** категории (дневник, ЛФК, приём, общее здоровье) — toggles.
3. Загрузка текущих правил из `reminder_rules` через API.
4. Сохранение при изменении → server action → обновление `reminder_rules`.

**Критерий:**
- Страница отображает текущие настройки.
- Изменения сохраняются.
- Каналы зависят от привязанных.

## Подэтап 12.2: API — CRUD правил напоминаний

**Задача:** серверные действия для управления напоминаниями.

**Файлы:**
- `apps/webapp/src/modules/reminders/`

**Действия:**
1. `getReminderRules(userId)` — текущие правила.
2. `updateReminderRule(ruleId, data)` — обновление расписания, каналов, категорий.
3. `createReminderRule(userId, data)` — если нет правила по категории.
4. Relay в integrator (через projection или прямой API) для синхронизации `user_reminder_rules`.

**Критерий:**
- Правила создаются/обновляются.
- Integrator получает обновления.

## Подэтап 12.3: Иконка колокольчика

**Задача:** колокольчик в шапке с пропущенными напоминаниями.

**Файлы:**
- `apps/webapp/src/shared/ui/PatientHeader.tsx`
- Модуль reminders

**Действия:**
1. Иконка 🔔 справа в шапке (перед меню).
2. Бейдж: количество непросмотренных напоминаний (из `reminder_occurrence_history` с `status != 'seen'`).
3. При нажатии: dropdown со списком пропущенных.
4. Каждое: текст напоминания + дата + кнопка «✓» (отметить просмотренным).

**Критерий:**
- Бейдж показывает счёт.
- Dropdown с пропущенными.
- Отметка просмотренным обнуляет бейдж.

## Подэтап 12.4: Статистика напоминаний

**Задача:** сколько просмотрел, сколько пропустил.

**Файлы:**
- Страница «Бот заботы» — блок статистики

**Действия:**
1. В нижней части страницы «Бот заботы»:
   - Всего отправлено: N.
   - Отмечено просмотренным: N (%).
   - Пропущено: N (%).
2. Агрегация из `reminder_occurrence_history`.

**Критерий:**
- Статистика корректна.
- Отображается на странице.
