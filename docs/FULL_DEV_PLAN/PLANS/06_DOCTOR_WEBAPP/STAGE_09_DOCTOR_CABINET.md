## Подэтап 9.1: Дашборд доктора

**Задача:** главная страница врача с плиткой метрик.

**Файлы:**
- `apps/webapp/src/app/app/doctor/page.tsx`
- `apps/webapp/src/modules/doctor-stats/`

**Действия:**

2. Плитки (карточки со счётчиками):

**Критерий:**

## Подэтап 9.6: Заметки врача

**Задача:** врач может оставлять заметки к клиенту/подписчику.

**Файлы:**
- Миграция: `apps/webapp/migrations/022_doctor_notes.sql`
- Компоненты карточки

**Действия:**
1. Миграция:
   ```sql
   CREATE TABLE IF NOT EXISTS doctor_notes (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id UUID NOT NULL REFERENCES platform_users(id) ON DELETE CASCADE,
     author_id UUID NOT NULL REFERENCES platform_users(id),
     text TEXT NOT NULL,
     created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
     updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
   );
   CREATE INDEX idx_doctor_notes_user ON doctor_notes(user_id, created_at DESC);

   ALTER TABLE platform_users ADD COLUMN IF NOT EXISTS is_blocked BOOLEAN NOT NULL DEFAULT false;
   ALTER TABLE platform_users ADD COLUMN IF NOT EXISTS blocked_at TIMESTAMPTZ;
   ALTER TABLE platform_users ADD COLUMN IF NOT EXISTS blocked_by UUID;
   ```
2. В карточке клиента/подписчика: блок «Заметки».
3. Кнопка «Добавить заметку» → текстовое поле.
4. Кнопка «Заблокировать подписчика» (с подтверждением).

**Критерий:**
- Заметки сохраняются и отображаются.
- Блокировка работает (заблокированный не может отправлять сообщения).
