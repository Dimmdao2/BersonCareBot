## Подэтап 7.1: Установка recharts

**Задача:** добавить recharts, настроить dynamic import.

**Файлы:**
- `apps/webapp/package.json`

**Действия:**
1. Если не установлен в этапе 4: `pnpm --filter webapp add recharts`.
2. Создать wrapper для dynamic import:
   ```tsx
   // modules/diaries/components/ChartWrapper.tsx
   'use client';
   import dynamic from 'next/dynamic';
   const LineChart = dynamic(() => import('recharts').then(m => m.LineChart), { ssr: false });
   ```
3. Проверить, что бандл не увеличивается для страниц без графиков.

**Критерий:**
- recharts загружается lazy.
- SSR не ломается.

## Подэтап 7.2: API агрегации

**Задача:** эндпоинты для данных графиков.

**Файлы:**
- Новые actions в `apps/webapp/src/modules/diaries/`

**Действия:**
1. `GET /api/patient/diary/symptom-stats?trackingId=X&period=week|month|all&offset=0`:
   - Агрегация `symptom_entries` по дням.
   - Возвращает: `[{ date, value, entryType }]`.
   - `offset` — смещение периода (0 = текущий, -1 = предыдущий).
2. `GET /api/patient/diary/lfk-stats?complexId=X&period=week|month|all&offset=0`:
   - Агрегация `lfk_sessions` по дням.
   - Возвращает: `[{ date, completed, duration, difficulty, pain }]`.
3. Оптимизация: индексы на `(tracking_id, recorded_at)` и `(complex_id, completed_at)` уже есть.

**Критерий:**
- API возвращает корректные агрегации.
- Период и offset работают.

## Подэтап 7.3: График симптомов

**Задача:** полноценный виджет статистики симптомов.

**Файлы:**
- Новый: `apps/webapp/src/modules/diaries/components/SymptomChart.tsx`

**Действия:**
1. Сверху: выпадающий список периода (неделя / месяц / всё время).
2. Слева и справа от списка: стрелки навигации (← →).
3. Ниже: если несколько симптомов — select «Выберите симптом». Если один — select не показывается.
4. График (recharts `<LineChart>`):
   - X: даты (точки).
   - Y: 0–10.
   - Линии разного цвета для разных симптомов (если будет мультивыбор).
   - Точки-маркеры на записях.
5. Если нет данных: «Нет записей за выбранный период».

**Критерий:**
- График отображает данные.
- Переключение периода и навигация работают.
- Мобильная адаптивность.

## Подэтап 7.4: Таблица/статистика ЛФК

**Задача:** обзорная и подробная таблицы ЛФК.

**Файлы:**
- Новый: `apps/webapp/src/modules/diaries/components/LfkStatsTable.tsx`

**Действия:**
1. Вкладка «Статистика» в дневнике ЛФК.
2. **Без выбранного комплекса** (или несколько) — обзорная таблица:
   - Колонки: Дата | Комплекс1 (✓/✗) | Комплекс2 (✓/✗) | ...
   - Названия комплексов — вертикально в заголовке.
3. **С выбранным комплексом** — подробная таблица:
   - Колонки: Дата | ✓ | Длительность | Сложность | Боль | Симптом (дневной) | Мин/Макс моментных | Кол-во отметок.
4. Данные из API подэтапа 7.2.
5. При большом количестве строк — виртуализация или пагинация.

**Критерий:**
- Обзорная таблица показывает все комплексы.
- Подробная таблица показывает метрики.
- Данные симптомов привязаны к дате.

## Подэтап 7.5: Мини-график на главной

**Задача:** обновить мини-статистику из этапа 4.4.

**Файлы:**
- `apps/webapp/src/modules/diaries/components/MiniStats.tsx`

**Действия:**
1. Подключить к реальным данным (API из 7.2).
2. Sparkline: первый отслеживаемый симптом за месяц.
3. Кружочки ЛФК:

**Критерий:**
- Мини-график отображает реальные данные.
- Кружочки ЛФК корректны.
- Клик → навигация.
