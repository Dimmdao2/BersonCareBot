## Этап 20: Мультитенантность и платежи

**Scope:**
- `tenant_id` во всех таблицах.
- Каталог специалистов: профили, фильтры, локация, стоимость.
- Модуль оплаты: ЮKassa или Тинькофф Оплата.
- Подписки: платные комплексы ЛФК.
- Бонусы: реферальная конверсия → бонус на счёт.
- Tenant isolation: каждый врач видит только свои данные.
- Супер-админ: управление всеми тенантами.

**Архитектурные решения:**
- Row-level security (RLS) в PostgreSQL ИЛИ application-level filtering.
- RLS предпочтительнее для безопасности, но сложнее в реализации.
- Рекомендация: начать с application-level `WHERE tenant_id = ?`, позже мигрировать на RLS.

**Платёжные системы (Россия):**
- **ЮKassa (YooMoney)** — самый популярный, хорошее API, SDK для Node.js.
- **Тинькофф Оплата** — хорошие тарифы, REST API.
- **CloudPayments** — хороший API, рекурентные платежи.
- Рекомендация: ЮKassa (наибольшее покрытие способов оплаты).
