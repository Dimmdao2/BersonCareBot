# Токены: `login_*` (вход) vs `link_*` (привязка канала)

| Префикс / сценарий | Назначение | Где обрабатывается | Секрет / TTL |
|--------------------|------------|-------------------|--------------|
| `login_<token>` | Вход в webapp через бота (Telegram `/start`) | `login_tokens` в webapp, polling `POST /api/auth/messenger/poll` | Хэш в БД, срок ~10 мин; см. `messenger/start` |
| `link_*` / channel-link | Привязка мессенджера к уже существующей сессии | Этап 3: `channel-link`, отдельные секреты в integrator | Не смешивать с login-токенами |

Интегратор при `/start login_...` должен вызывать webapp/бота для **подтверждения login** (отдельный контракт от `link_*`). Детали сценария — этап **5.4** в `PLAN.md`.
