## Webapp Клиент

### 1. Тех стек

1. Вероятно надо бы найти какой-то готовые виджеты для графиков и таблиц для статистики и журналов.
2. Надо сделать нормально систему авторизации:
   1. Обеспечить возможность создания пароля, сброса пароля.
   2. Вход по паролю будет доступен только в веб странице или PWA — в мессенджерах априори первичная авторизация по ид чата + возможность прикрутить телефон, имэйл и другой мессенджер.
   3. Это надо для экономии моих денег за рассылку смс.
   4. Также стоит подумать о длительном хранении данных сессии при работе в браузере.
3. Переписка:
   1. Надо продумать хранение сообщений, возможность прикреплять файлы и фотографии к переписке в приложении.
   2. Надо продумать копирование сообщений (в тч голосовых) в историю переписки вебапп из чатов — при общении в чате.
4. Приложение:
   1. Надо продумать и запланировать создание PWA или Cordova App — что для этого потребуется. Будет ли PWA выглядеть как приложение без строки браузера, будет ли храниться кэш страниц на устройстве.
   2. После создания PWA/Cordova подключим пуш уведомления.
   3. После подключения пуш уведомлений они смогут стать каналом отправки сообщений и уведомлений в настройках уведомлений.
5. Ремайндер:
   1. Надо уже начать реализовывать напоминалки.
   2. Кнопка «Бот заботы» в главном меню клиента в блоке «Кабинет» ведет на страницу настроек напоминалок:
      1. Выбор каналов напоминаний.
      2. Выбор расписания и о чем напоминать.
      3. Статистика — сколько раз клиент отметил что послушал напоминание, а сколько — забил.
   3. В шапке приложения у клиента — иконка колокольчика справа (около меню «три полоски») — там будут открываться пропущенные.

### 2. Найденные проблемы

1. Призыв подключить еще один мессенджер — при переходе по кнопке Max бот открывается, но привязка не происходит и никакой реакции в боте в Макс тоже не происходит.
2. При нажатии на кнопку «изменить» около телефона — ничего не происходит (точнее просто мигает страница — видимо перезагрузка).
3. Пропала кнопка «задать вопрос» после привязки мессенджера телеграм. Вероятно, ее показ фильтруется наличием мессенджера у юзера (а должен — наличием секретного ключа в переданной сессии), либо же при открытии вебапп в мессенджере ключ сессии телеграм передается и в системный браузер. Надо сделать так чтобы ключ не сохранялся при закрытии вебапп в телеграм / макс и не передавался в другую сессию в другом браузере. То есть пока я в вебапп — кнопки нет. Пока я в браузере — есть.
4. Еще про кнопку «Задать вопрос». Сейчас она мигает при перезагрузке страницы. Вероятно она сначала установлена как «Видимая», а при условии фильтрации скрывается. Надо наоборот: изначально — невидимая, а при отсутствии ключа сессии мессенджера — появляется.

### 3. Верстка и дизайн — что нравится и надо сохранить

1. Общий стиль уже хорош.
2. Иконка домика.
3. Стрелочка назад.
4. Стиль информационного сообщения на экране «Настройки уведомлений» с текстом «Настройка каналов уведомлений будет доступна в ближайшем обновлении. Сейчас уведомления приходят во все подключённые каналы.» Его надо использовать для информационных блоков для привлечения внимания (далее — «информационный блок»).

### 4. Верстка и дизайн — что плохо

1. Слишком крупно — большие блоки, пустые пространства. Где-то большие отступы между блоками, где-то маленькие.
2. Надо слегка уменьшить шрифт заголовков.
3. Шапка слишком толстая — надо уменьшить процентов на 15.
4. Правое Меню ок, но справа отступ великоват — сделать как слева.
5. Кнопки на страницах разного размера, верстка едет.
6. Скругление у кнопок слишком сильное, конфликтует с полями ввода и углами блоков.
7. Все кнопки одноцветные. Надо заложить в шаблон пресеты кнопок:
   1. Обычная.
   2. Инверсия — темно серая с белым фоном.
   3. Синяя (серовато темно-синий теплый, не яркий, с белым текстом на ней).
   4. Красная — где требуется внимание к действию — по аналогии с синим.

### 5. Шапка

1. В шапку надо перенести заголовок страницы: по центру более мелким и менее ярким шрифтом.
2. Иконка «дом» уезжает влево к стрелке назад.
3. Справа перед тремя полосками меню добавляется иконка сообщения. Если есть непрочитанные — иконка подсвечивается (или на иконке появляется красная точка справа сверху). При нажатии открывается диалог.

### 6. Боковое Меню

1. Добавить пункт «Адрес кабинета» — пока пусть открывает в новой вкладке страницу с адресом кабинета (ссылку указал в инструкции к «Мои записи»), потом добавим страницу внутри приложения.
2. Надо добавить пункт «Сообщения». Пусть там хранится история переписки во всех привязанных чатах и в вебе.
3. Вернуть пункт «Справка» и создать такую страницу.
4. Можно добавить пункт «Поделиться с другом»:
   1. При нажатии на кнопку ссылка должна копироваться в буфер обмена, и всплывающее сообщение — «ссылка скопирована в буфер обмена».
   2. Ссылка должна генерироваться с референсом, уникальным для данного юзера.
   3. Надо сохранять в базу историю всех пришедших по этой ссылке. В будущем при появлении платной подписки за каждую первую покупку подписки пригласившему юзеру будет падать бонус на счет.
5. Было бы классно сделать пункт «Установить приложение». Пока открывает страницу с инструкцией, как «Сохранить на рабочий стол» — подойдет для PWA. Потом будут ссылки на приложение в магазине.

### 7. Главная страница (кабинет, дом)

Блоки на странице:

1. **Кабинет:**
   1. Дневник
   2. Мои записи
   3. Мои комплексы ЛФК (создаются и назначаются врачом либо покупаются готовые) — пока скрытый пункт
2. **Уроки:**
   1. Скорая помощь
   2. Разминки
   3. Тренировки
   4. Полезные материалы
3. Ниже блок **Важные новости** — выделяется как информационный блок, или при статусе «важно» — красным. Показывается и скрывается только специалистом, у него же в кабинете ведется статистика — сколько человек просмотрело (открывали кабинет в момент наличия новости).
4. Ниже блок с **Уведомления**. Показываются рассылки, на которые подписан юзер в любом канале. Под рассылкой внизу кнопочка, чтобы отметить что «просмотрено» — просмотренные скрываются.
5. Блоки рассылки и новости видны только при наличии активного сообщения.
6. Ниже — Блок с **мотивашкой** или какой-нибудь шуткой (без названия, выделен легким приятным цветом в гамме приложения). Внутри блока надпись (без заголовка), меняется рандомно каждый день. Мотивации буду добавлять в кабинете доктора.
7. Ниже — **Статистика:**
   1. Упрощенный график симптомов за последний месяц (без обозначения дат и баллов).
   2. Если наблюдение меньше месяца — то за существующий период.
   3. Под линией времени — кружочками отмечено наличие записей о выполненных комплексах ЛФК:
      1. Зеленый — выполнен.
      2. Серый — в расписании нет занятия в этот день.
      3. Коричнево-желтый — пропуск.
   4. Если нет привязки к номеру телефона — полупрозрачный график-заглушка с надписью на плашке по центру графика «Статистика доступна зарегистрированным пользователям при ведении дневника». График-заглушка должен идти вверх.

### 8. Профиль: личные данные

1. Надо сделать и ФИО и телефон и имэйл — однотипно по подобию телефона — простым текстом.
2. Сейчас недоступна привязка имэйл — надо сделать (следующий шаг — настройка мэйлера в интеграциях интегратора).
3. Кнопки «Изменить» справа от данных (у правого края блока), однотипные, некрупные, в одну колонку.
4. При нажатии текст меняется на поле ввода, в котором уже подставлено текущее значение.
5. Если значения нет (пустой имэйл, нет телефона) — видим только кнопку «добавить» справа от заголовка (от слов «телефон» или «email»). Под заголовком данных при этом нет. При нажатии становится виден инпут и справа кнопка «привязать», кнопка «добавить» на этот период пропадает. Состояние блока сбрасывается при переходе в другое окно.
6. При изменении значения в инпуте имэйла или телефона и нажатии кнопки «привязать» снизу под инпутом появляется еще один инпут для ввода кода подтверждения. Одноразовый код отправляется по данному каналу связи через интегратор.
7. Надо проверить, как долго хранится сессия на мобильном. В идеале — не сбрасывать ее вообще, чтобы не тратить смс-ки.
8. Добавить возможность создания пароля в кабинете, чтобы по желанию авторизоваться по нему.
9. В авторизацию по номеру телефона и имейлу надо сделать возможность отправить код повторно и обязательно таймер на отправку не чаще чем раз в минуту. Через три попытки — блокировать на 10 минут.

### 9. Профиль: Привязанные каналы

1. Убрать пока иконки слева от названия мессенджера.
2. Подключение мессенджеров: При нажатии на кнопку «подключить» (макс, тг, в будущем вк): надо формировать временный секрет и передавать его мессенджеру в строку /start например `?start=link_<secret_link_key>` и автоматически привязывать без запроса номера телефона. Сценарий по этому шаблону команды определяет команду привязки, проверяет ключи и если ок — привязывает и сбрасывает секрет. Команда `?start=` точно работает для ТГ, надо проверить возможности Max и ВК. Изучи их API и напиши как это можно сделать.

### 10. Настройки уведомлений

1. Заголовок «Подписки на уведомления».
2. Надо дать возможность выбирать также «смс», если привязан номер телефона.
3. Возможность выбирать имэйл как канал.
4. Надо под блоком списка уведомлений добавить блок привязки мессенджеров — такой же как в Профиле (переиспользовать шаблон, а не копировать). И блок привязки имэйла — переиспользовать тоже.

3. Бессмысленный заголовок «Ближайшие записи».
4. При отсутствии записей:

   2. Надо:

   6. Статус запроса на перенос записи — другим цветом — вероятно сиреневым.

### 12. Дневник

1. Объединить в один пункт меню «Дневник».
2. В верхней части страницы две вкладки: «Симптомы» и «ЛФК» — для быстрого перехода между страниц.
3. Для расширения функционала, в том числе под мультитенантную систему (тенант — терапевт) вероятно нужно будет сделать настройки дневника с выбором отображаемых дневников. То есть стартово: нет дневников, кнопка «Добавить дневник» и информационный блок «У вас нет дневников для отслеживания». Для выбора открывается справочник типов дневников.
4. Тк это дневники — специфичные сущности их надо будет добавлять отдельно как механизмы со своей бизнес-логикой, возможно — перекрестной статистикой и тд. То есть дневники — это отдельный цельный модуль внутри приложения, как и, например, ремайндер.
5. Впоследствии, вероятно, добавляем дневники (это пока просто идеи):
   1. Общее здоровье / нутрициология: Вес, КБЖУ, метрики сна, заметки по самочувствию.
   2. Психология: эмоции / мысли / реакции / дневник благодарности.

### 13. Дневник симптомов

1. Не выделяется цветом кнопка балла симптома при выборе (нажатии). В идеале для каждой цифры свой цвет — постепенное изменение от зеленого на нуле и до темнобордового на 10ке.
2. Кнопки выбора балла — сейчас не круглые, надо сделать фикс круг.
3. После сохранения должно появляться всплывающее сообщение (в прямоугольном попапе) «Запись добавлена» — на секунду появляться и растворяться.
4. Нужна возможность удалять (помечать удаленным) / архивировать / редактировать название симптома (иконка три точки справа от названия в таблице, при нажатии — всплывающее меню с названием симптома и выбором действия).
5. Поменять местами порядок блоков:
   1. Добавить запись
   2. Отслеживаемые симптомы
   3. Статистика
6. Добавление симптома:
   1. Помимо названия (опционально) есть поля:
   2. Тип симптома — заполняется из справочника: боль / жжение / онемение / слабость / напряжение / отек / ограничение подвижности / кинезиофобия / тревожность / паническая атака и тд. Справочник общий по терапевту. Терапевт в своем кабинете может добавлять значения в справочник.
   3. Регион (шея, поясница, плечо и тд) — поле с выбором, варианты значения подгружаются из справочника.
   4. Справа от региона — кнопки «прав» и «лев», опционально.
   5. Диагноз. Ввод возможен как свободным вводом, так и из справочника — справа кнопка с иконкой справочника (поначалу загрузить туда основные диагнозы ортопедических заболеваний и периферических нейропатий).
   6. Стадия (период): выбирается из справочника (пункты надо обсудить и улучшить / почистить список):
      1. Острый период
      2. Заживление
      3. Ремоделирование тканей
      4. Адаптация (стабилизация состояния)
      5. Хроническое течение
      6. Восстановление функций (бытовая адаптация)
      7. Профилактика обострений (укрепление)
      8. Возврат в спорт (подготовка к тренировочной активности)
      9. Улучшение спорт результатов (спортспецифичный тренинг)

### 14. Виджет статистики (дневник симптомов)

1. Выбор вида (выпадающий список): неделя / месяц / все время.
2. Справа и слева от поля выбора вида стрелки — листаем недели / месяцы.

### 11. Комплекс упражнений (ЛФК)

Режим просмотра и режим создания/редактирования (Конструктор комплекса). Комплекс содержит:

1. Название.
2. Назначение (краткое описание).
3. Регион.
4. Диагноз — вероятно надо привязку комплекса к диагнозам делать отдельной таблицей — может быть несколько комплексов на один диагноз и наоборот.
5. Стадия (период) — привязано к диагнозу (тот же справочник что и при добавлении симптомов в дневнике).
6. Уровень сложности комплекса от 1 до 10.
7. Назначение — для острого состояния, хронической боли и тд. Выбор из справочника.
8. Выбор упражнений из списка — попап со списком упражнений.
   1. Попап со списком упражнений открывается почти на весь экран, но по краям есть затемненные поля и видно, что внизу — конструктор справочника.
   2. Попап на высоту экрана, то есть не прокручивается.
   3. Наверху: заголовок «Добавление упражнений».
   4. Под заголовком строка поиска по названию и фильтрам, как на странице справочника:
      1. Региона.
      2. Типа упражнения.
      3. Диагноза.
      4. Стадии.
   5. В списке в строке с упражнением видим:
      1. Название — заголовок строки.
      2. Внизу: регион, тип, теги.
      3. Справа — сложность в цифрах.
      4. Совсем справа — флаг выбора (галочка).
   6. Под списком упражнений внизу блока кнопки (расположены горизонтально):
      1. Добавить выбранные.
      2. Создать новое.
   7. Можно отметить в списке упражнений галочками нужные строки, потом изменить фильтры или ввести в поиск новое название — выбранные упражнения, которые не подходят под фильтры станут не видны, но состояние выбора не сбросится. При нажатии на кнопку «добавить выбранные» все подгрузятся в комплекс ЛФК, попап закрывается, и продолжаем редактировать комплекс в конструкторе.
   8. При выборе «добавить новое» — открываем попап добавления упражнения в систему. По принципу попапа со списком упражнений — внизу вместо кнопки «сохранить» — кнопка «добавить в комплекс». При нажатии попап закрывается и упражнение добавляется в конструктор комплекса.
   9. Вероятно стоит переиспользовать блоки конструктора упражнений и списка упражнений, чтобы не плодить дубли и легче их менять при добавлении / изменении полей, названий и тд.
9. Упражнения в комплексе указаны в порядке добавления.
10. Упражнения могут дублироваться — то есть одинаковая растяжка может быть после двух разных силовых упражнений в разных участках комплекса.
11. Можно менять порядок (либо стрелочками вверх и вниз справа, а в идеале — драг энд дроп с перетаскиванием по списку).
12. Для каждого упражнения указание количества повторений / ПМ и подходов.
13. Для унилатеральных упражнений отдельно указание для прав и лев.
14. Подробное описание, комментарии к комплексу.
15. Противопоказания.
16. Доп рекомендации (опционально):
    1. Расписание выполнения (по дням недели, или сколько раз в период).
    2. Утро / день / вечер.
    3. Максимальный балл боли при выполнении на весь комплекс (отображается для тех упражнений где нет установленного значения).
    4. Текстом свободно доп. рекомендации / комментарии.

### 12. Добавление комплекса ЛФК пациенту

1. Стадия / цели (острая боль, хроническая боль, профилактика обострений, укрепление региона, подготовка к спорту и тд — справочник).
2. Этап реабилитации (номер или текстом), возможно в будущем справочник по этапам.
3. Название комплекса — либо произвольно (текстовое поле), либо выбор конкретного комплекса из базы — тогда подставляется его название и привязывается готовый комплекс.
4. Править блок информации (привязывается только для данного индив. назначения, перекрывают значения по умолчанию для комплекса):
   1. Расписание выполнения (по дням недели, или сколько раз в период).
   2. Дополнительные рекомендации по выполнению для клиента (опционально):
      1. Утро / день / вечер.
      2. Максимальный балл боли при выполнении на весь комплекс (отображается для тех упражнений где нет установленного значения).
      3. Текстом свободно доп. рекомендации / комментарии.

### 13. Добавление индивидуального комплекса клиенту

На экране назначения комплекса клиенту должны быть вкладки:

1. **Список готовых комплексов** — с фильтрацией по регионам, диагнозам, стадиям (периоду), тегам, уровню сложности.
2. **Конструктор комплекса ЛФК:** Аналогично созданию обычного комплекса.
   1. Название.
   2. Загрузить из готового комплекса для правки.
   3. Удалять и добавлять упражнения, менять порядок, кол-во повторений и подходов.
   4. Добавлять к каждому упражнению комментарий специфичный этому комплексу.
   5. Добавлять максимальный балл боли.
   6. Все доп.данные (сложность / диагноз / стадия / описание) — опционально если назначение индивидуальное.
   7. Возможность сохранить как готовый (общедоступный) комплекс — черновик.
   8. Для публикации готового комплекса из черновика — все поля данных становятся обязательными.
   9. Индивидуальные комплексы в списке выбора комплексов отображаются в отдельной вкладке с указанием кому и когда назначены.
3. Расписание выполнения (по дням недели, или сколько раз в период).
4. Дополнительные рекомендации по выполнению для клиента (опционально):
   1. Утро / день / вечер.
   2. Максимальный балл боли при выполнении на весь комплекс (отображается для тех упражнений где нет установленного значения).
   3. Текстом свободно доп. рекомендации / комментарии.
5. Кнопка «Назначить» — комплекс появляется у клиента в экране Дневника и в экране «Мои комплексы ЛФК».

### 14. Список комплексов

1. **Готовые комплексы (опубликованные)** — список:
   1. Название.
   2. Краткое описание.
   3. При нажатии на комплекс две вкладки:
      1. **Информация:**
         1. Блок настроек:
            1. Стоимость.
            2. Снять с публикации / опубликовать.
         2. Метрики комплекса:
            1. Назначен клиенту.
            2. Куплен онлайн.
            3. Выполнен по дневникам.
            4. Процент пропусков (не отмечен в дневнике относительно расписания).
      2. **Состав комплекса:**
         1. Упражнения и все данные — как в пункте «Комплекс упражнений (ЛФК)».
      3. Править / архивировать можно только снятый с публикации комплекс.
      4. Даже при снятии комплекса он не меняется у клиентов: тех кому он был назначен на приеме или кто его купил.
      5. Возможно нужна отдельно сделать механизм отзыва / форс-правки для улучшения упражнений, исправления возможных ошибок. Надо продумать как это делать.
