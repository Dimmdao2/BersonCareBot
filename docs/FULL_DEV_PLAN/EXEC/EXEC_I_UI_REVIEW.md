## I.1 — Кнопки: единая система (пп. 22, 33, 34, 35, 50, 55)

### Проблема

Кнопки по приложению неоднородны: разное скругление, размеры, цвета текста, реакция на нажатие. Синие кнопки в кабинете хорошие, но в других местах — старые скруглённые. Текст на кнопке «Привязать Max» чёрный на синем фоне. Нажатие почти не ощущается.

### Что сделать

1. **Унифицировать Button компонент** (`apps/webapp/src/components/ui/button.tsx`):
   - Уменьшить `border-radius` примерно вдвое от текущего (было слишком скруглённо).
   - Для `variant="default"` (синяя): фон синий, текст **белый**, при нажатии (`active:`) — темнее + `shadow-inner` (утопленный вид).
   - Для всех вариантов: добавить `active:` состояние — затемнение + внутренняя тень.
   - Единая высота, padding, размер шрифта для всех кнопок одного размера.
   - Проверить ВСЕ места использования Button — нигде не должно быть переопределения текстового цвета на чёрный поверх синего фона.

2. **Аудит всех кнопок в приложении**: пройти по страницам, найти кнопки со старым стилем (класс `button button--*` из globals.css), заменить на `<Button>` из shadcn.

3. **Все нажимаемые элементы** (FeatureCard, ссылки-кнопки, карточки в списках) — добавить `active:scale-[0.98]` или аналогичный визуальный отклик.

## I.3 — PIN-код: банковский ввод (п. 1)

### Проблема

PIN сейчас — обычный `<Input type="password">`. Должен быть как банковский: 4 отдельных квадрата, каждый под одну цифру, строго 4 цифры.

### Что сделать

**Файл**: `apps/webapp/src/shared/ui/auth/PinInput.tsx`

1. Заменить единственный `<Input>` на 4 отдельных `<input>`:
   - Каждый: `w-12 h-14 text-center text-2xl font-bold border rounded-lg`, `maxLength=1`, `inputMode="numeric"`.
   - При вводе цифры — автофокус на следующее поле.
   - При Backspace — фокус на предыдущее.
   - При paste в первое поле — распределить все 4 цифры по полям.
   - Визуально: горизонтальный ряд с `gap-3`.

2. Убрать `maxLength={6}` — PIN строго 4 цифры.

3. Валидация: `/^\d{4}$/` вместо `/^\d{4,6}$/`.

4. Auto-submit: при вводе 4-й цифры — автоматически вызвать `onSubmit`.

## I.8 — Статистика: переключатели, график, журнал (пп. 23, 24, 25, 26, 27, 28, 29, 30, 31, 46, 48, 49, 53)

### Проблема

Переключатели периода невнятные. «Всё» показывает бесконечный диапазон. Даты на графике в формате 03/21. График плохо виден. Журнал показывается прямо под графиком. Подписи обрезаны.

### Что сделать

1. **Переключатели «Неделя / Месяц / Всё»**:
   - Использовать shadcn `ToggleGroup` или кастомный: активный — `bg-primary text-primary-foreground`, неактивный — `bg-muted text-muted-foreground`.
   - Должно быть очевидно, какой период выбран.

2. **Период «Всё»**: ограничить слева датой первой записи пользователя. Запрос: `SELECT MIN(recorded_at) FROM symptom_trackings WHERE user_id = $1`.

3. **Формат дат на оси X графика**:
   - Неделя: `ПН`, `ВТ`, `СР`, `ЧТ`, `ПТ`, `СБ`, `ВС`.
   - Месяц: 4 опорных даты (начало недель), формат `1 мар`, `8 мар`, `15 мар`, `22 мар`.
   - Всё: месяцы (`янв`, `фев`, `мар`...).

4. **Ось Y**: убрать дублирующие серые цифры 0 и 10 слева, если шкала и так понятна. Оставить только деления без текста или минимально (0, 5, 10).

5. **Видимость графика**: увеличить `strokeWidth` линии до 2–3px. Точки данных — `r={4}` вместо `r={2}`. Цвет линии — более насыщенный.

6. **Подписи под графиком**: проверить что контейнер имеет достаточный `pb` (padding-bottom), чтобы текст не обрезался.

7. **Журнал — отдельный экран**:
   - Под графиком/статистикой — кнопка или ссылка «Открыть журнал» (не сам журнал).
   - По нажатию — переход на `/app/patient/diary/symptoms/journal` (или модал/sheet).
   - Журнал открывается на тот же период, что выбран в статистике.
   - Фильтр по диапазону дат сверху журнала.
   - Максимальный период в журнале — месяц (пагинация по месяцам).
   - Каждая запись: меню «⋯» → «Редактировать» / «Удалить».

8. **Единая логика для симптомов и ЛФК**: переключатели, журнал, фильтры, действия — одинаковые в обоих дневниках.

9. **Отступы**: между графиком и следующим блоком — `mt-6` минимум.
