## Особенность Max: нет reply keyboard

**Проблема:** Max не поддерживает reply keyboard (постоянные кнопки внизу чата). У Telegram через reply keyboard пользователь видит кнопки «Дневник», «Записи», «Кабинет» и т.д. прямо под полем ввода.

**Решение для Max:** inline-меню в боте должно компенсировать отсутствие reply keyboard:

| Элемент | Telegram (reply keyboard) | Max (inline keyboard) |
|---------|--------------------------|----------------------|
| Дневник | Кнопка в reply keyboard | Кнопка в inline-меню бота |
| Записи | Кнопка в reply keyboard | Блок в inline-меню: если есть активные записи — список, иначе кнопка «Записаться» |
| Кабинет (webapp) | Кнопка в reply keyboard → Mini App | Кнопка `open_app` (или `link`) → webapp |
| Настройки | Через reply keyboard / webapp | Кнопка в inline-меню |

**Правило для bot-режима webapp:** раз у Max нет reply keyboard → на главной в bot-режиме должны быть и кнопка «Дневник», и блок записей (ближайших, если есть).

**Изменения в конфиге:**

```
patientHomeBlocksByPlatform.bot:
  БЫЛО:  ["materials", "assistant", "purchases", "lfk-complexes", "patient-card"]
  СТАЛО: ["diary", "appointments", "materials", "assistant", "purchases", "lfk-complexes", "patient-card"]
```

Также inline-меню бота в integrator (Max adapter) должно содержать:
- «📖 Дневник» → webapp diary page
- «📋 Мои записи» → webapp appointments page (или список, если есть активные)
- «🏠 Кабинет» → webapp home
- «⚙ Настройки» → webapp settings

## Зависимости между фазами

```
A (багфиксы + pre-prod) ─────────┐
                                  ├──→ B (дизайн-система) ──→ D (главная + записи)
                                  │         │
                                  │         └──→ C (профиль + auth)
                                  │
                                  └──→ A.2 (durable dispatch) ──→ K (напоминания + интеграции)

B (дизайн-система) ──→ E (дневники) ──→ F (графики)
                   ──→ G (сообщения)
                   ──→ H (кабинет врача)

E (дневники) + I (CMS) ──→ J (ЛФК модуль)

H (кабинет врача) + K (интеграции) ──→ L (настройки/админ)

L ──→ M (post-prod)
```

**Параллелизм:** после B можно запускать C, D, E, G, H параллельно (с разными агентами).

## Ключевые правила для агента-исполнителя

1. Каждая задача = один PR, `pnpm run ci` зелёный.
2. Не менять существующие миграции, только добавлять новые (идемпотентные).
3. Все UI-тексты на русском.
4. Новые компоненты в `modules/<name>/components/`, не в `shared/ui/`.
5. Tailwind 4 + shadcn/ui для всего нового. Старый CSS переводить при касании.
6. Медиа → S3. Новые таблицы с `tenant_id` (nullable).
7. Серверные компоненты по умолчанию, `"use client"` только для интерактива.
8. Документацию синхронизировать с реальностью.
9. **Max bot-режим**: учитывать отсутствие reply keyboard — все ключевые действия (дневник, записи) должны быть доступны через inline-меню бота и через блоки на главной webapp.
