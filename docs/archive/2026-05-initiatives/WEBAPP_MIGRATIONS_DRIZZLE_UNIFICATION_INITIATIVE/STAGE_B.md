# STAGE_B — Drizzle Consolidation (Codex)

## Цель

Сконсолидировать webapp schema evolution в Drizzle без регрессий для существующих БД.

## Шаги

1. По результатам Stage A перенести в Drizzle path **приоритетные** пробелы канонического пути (в первую очередь объекты, которые одновременно **guardrail/deploy-critical** и отсутствовали в Drizzle-SQL до этапа).
2. Проверить идемпотентность на уже мигрированной БД.
3. Зафиксировать фактическое состояние **greenfield/bootstrap**: при noop `0000` и отсутствии merged baseline полная схема `public` одним только Drizzle migrate **может не воспроизводиться** — это residual с явной политикой (`migrate:legacy` / ops), а не скрытый провал этапа.
4. Зафиксировать совместимость и ограничения (если остаются).
5. Обновить docs инициативы и `LOG.md`.

## Выход этапа

- **закрыты** согласованные на этапе пробелы Drizzle path (в закрытой инициативе: guardrail для `recommendations.domain` — миграция `0053`, плюс выравнивание post-migrate проверки для webapp-only deploy);
- явно документировано, какие области Stage A по-прежнему зависят от legacy / исторического состояния БД и почему полный перенос всего runtime-слоя в Drizzle **не заявлен** узким завершением Stage B.

## Gate закрытия

- целевые миграционные проверки успешны;
- нет блокирующих замечаний по риску повторного DDL;
- в `LOG.md` есть итог этапа B.
