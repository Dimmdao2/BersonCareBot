# CONTENT_PLAN — Patient Home Redesign

Контентный план для Phase 0 инициативы `PATIENT_HOME_REDESIGN_INITIATIVE`.

Статус: черновик для заполнения CMS и настройки главной.

Важно: этот файл **не является источником хардкода**. Runtime-источник главной страницы — БД CMS:

- `content_sections` хранит сами разделы, их `icon_image_url` и `cover_image_url`;
- `patient_home_blocks` хранит видимость и порядок блоков главной;
- `patient_home_block_items` хранит, какие материалы/разделы/курсы добавлены в каждый блок, их порядок и overrides.

Slug-и ниже — рекомендуемые стабильные идентификаторы для CMS, чтобы не создавать хаотичные URL. Их можно изменить до начала заполнения CMS.

Для агентов-исполнителей: **не переносить эти slug-и в runtime-код**. Не делать списки, switch/case, маппинги или специальные UI-ветки по значениям из этого файла. Если в тестах нужны slug-и, используйте их только как локальные fixture-данные для проверки generic-поведения.

Видео/иконки/обложки должны загружаться через CMS-медиатеку и затем выбираться в полях разделов/материалов или в настройке блоков главной.
