# Risk register — HLS dual delivery

Идентификаторы рисков стабильны для ссылок из PR и postmortem.

**Связь:** [00-master-plan.md](./00-master-plan.md) · смягчение в rollout: [03-rollout-strategy.md](./03-rollout-strategy.md)

| ID | Риск | Вероятность | Влияние | Митигация | Остаточный риск |
|----|------|-------------|---------|-----------|-----------------|
| R1 | Регрессия `GET /api/media/[id]` (401/302) для существующих страниц | Средняя | Высокая | Не менять семантику маршрута до согласования; отдельный playback JSON; регрессионные тесты | Низкий |
| R2 | FFmpeg грузит CPU и мешает webapp на одном хосте | Высокая (v1) | Средняя | `nice`, 1 concurrent job, отдельный systemd slice, план вынести worker | Средний |
| R3 | Частичный upload HLS → клиент ломается на плохом master | Средняя | Средняя | Атомарность: выставлять `hls_ready` только после verify manifest + N сегментов | Низкий |
| R4 | Presigned URL для каждого сегмента — всплеск нагрузки на signer | Средняя | Низкая | Кэш на клиенте; короткий плейлист; в фазе 09 — session policy | Низкий |
| R5 | Safari vs Chrome различия (CORS, native HLS) | Средняя | Средняя | Явные тесты на Safari; hls.js fallback policy | Средний |
| R6 | Утечка S3 URL в логи браузера/серверов | Низкая | Средняя | Не логировать query string presigned; redact в pino | Низкий |
| R7 | Backfill переполняет диск tmp worker | Средняя | Высокая | Stream download/upload; чистка tmp; лимит параллелизма | Низкий |
| R8 | Путаница с integrator worker | Низкая | Высокая | Документация и код-ревью: не использовать projection queue | Низкий |
| R9 | Долгие миграции БД на большой `media_files` | Низкая | Средняя | Add column nullable + индексы CONCURRENTLY при необходимости | Низкий |
| R10 | Watermark (фаза 10) удваивает время транскода | Средняя | Средняя | Отдельный флаг; только для чувствительного контента | Средний |
| R11 | Заказчик ожидает `apps/api` (Fastify) — команда ищет не там | Низкая | Низкая | Явно в master-plan и gap: API = webapp routes | Низкий |

**Процесс:** перед фазами 06–08 — пересмотр регистра; закрытые риски не удалять, помечать датой.
