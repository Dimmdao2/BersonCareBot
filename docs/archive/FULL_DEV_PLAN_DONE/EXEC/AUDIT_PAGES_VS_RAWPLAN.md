## Покрытие vs таблица EXEC H.5.2 / «все страницы»

**Сверено построчно с RAW §7–18 (patient) и ключевыми экранами doctor из RAW:** главная, профиль, каналы, уведомления, кабинет записей, дневник (вкл. симптомы/ЛФК агрегатом), сообщения, «задать вопрос», быстрое добавление, дашборд врача, сообщения врача, клиенты, подписчики, CMS.

**Не разобраны отдельным подпунктом** (можно добавить во вторую итерацию аудита, если нужен полный матричный учёт каждого `page.tsx`):

- Patient: `/app/patient/emergency`, `/lessons`, `/content/[slug]`, `/bind-phone`, `/help`, `/install`, `/purchases`, `/reminders` и др. вспомогательные маршруты.
- Doctor: `/app/doctor/appointments`, `/broadcasts`, `/stats`, `/references`, `/exercises`, `/lfk-templates`, `/settings`, `/content/news` и т.д. — частично попадают в обобщённый блок «D-REF-01 backlog», без построчного RAW.

Итог: для **цели H.5** (отклонения по основным экранам из RAW §7–18 и doctor §5–9) покрытие **достаточное**; **полный** перечень всех `patient/**/page.tsx` и `doctor/**/page.tsx` в документе **не** выверялся построчно — зафиксировано как пробел при необходимости расширения.
