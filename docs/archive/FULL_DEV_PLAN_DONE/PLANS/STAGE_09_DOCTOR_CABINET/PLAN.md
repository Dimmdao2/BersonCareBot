## Для агента в режиме «авто»: как пользоваться этим документом

1. **Выполнять один микрошаг за прогон** (например, только `9.1.2`), если явно не сказано объединить несколько шагов.
2. **Перед кодом (обязательно):**  
   - `docs/ARCHITECTURE/SERVER CONVENTIONS.md`  
   - `README.md`, `docs/README.md`  
   - Текущие реализации: `apps/webapp/src/modules/doctor-cabinet/`, `doctor-clients/`, `doctor-appointments/`, `doctor-stats/`, страницы под `apps/webapp/src/app/app/doctor/`.
3. **Номер миграции:** взять **следующий свободный** после максимального `NNN_*.sql` в `apps/webapp/migrations/` **на момент начала работ** (сейчас в репозитории последний **`025_*`** → новые миграции этапа 9 начинать с **`026_*`** и далее по одной на логический блок).  
   **Запрещено** переиспользовать номер **`022_*`** под `doctor_notes` — файл **`022_reference_tables_and_seed.sql` уже существует**.
4. **Паттерны webapp:** `buildAppDeps` как composition root; страницы врача через `requireDoctorAccess` из `app-layer/guards`; API под `/api/doctor/*` с проверкой роли врача/админа; SQL в `infra/repos` или сервисах, не 100+ строк копипастой в `page.tsx`.
5. **Жёсткие запреты (весь этап 9):**
   - не ослаблять авторизацию на `/api/doctor/*` и затрагиваемых `/api/patient/*` «для тестов»;
   - не выдумывать пути, имена таблиц и env — сверять с `SERVER CONVENTIONS.md` и существующими миграциями;
   - не добавлять таблицу `patient_card` без отдельной миграции и плана — в MVP список «Клиенты» опирается на **`appointment_records`** (и при необходимости на существующие поля `platform_users`), пока в плане не появится явная схема карт;
   - не писать секреты в репозиторий;
   - новые npm-зависимости — только с обоснованием и прохождением `pnpm run ci`;
   - admin-only UI и API: **никогда** не опираться только на скрытие кнопок — проверка роли на сервере обязательна.
6. **Тесты:** для каждого шага, где меняется или добавляется **поведение** (сервис, репозиторий, API, чистая функция), — unit/integration `*.test.ts` рядом по принятому в проекте образцу. **E2E in-process** (`apps/webapp/e2e/`) — минимум один новый или расширенный файл на завершённый крупный блок (9.1–9.7) + обновление при изменении навигации.
7. **Финальная проверка этапа 9:**  
   `pnpm install --frozen-lockfile && pnpm run ci` и `pnpm --dir apps/webapp run test:e2e` (см. `.cursor/rules/pre-push-ci.mdc`).

```sql

  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),

CREATE INDEX IF NOT EXISTS idx_doctor_notes_user ON doctor_notes(user_id, created_at DESC);
```
