## 2. Найденные проблемы

### Критичные

- **`apps/webapp/src/infra/repos/pgDoctorAppointments.ts`** — метод `getDashboardAppointmentMetrics`: подсчёт отмен за месяц использует `COUNT` по `status = 'canceled'` без исключения `last_event` в `('event-remove-record', 'event-delete-record')`, в отличие от `getAppointmentStats`. Риск повторного включения «удалений» в метрику отмен. Относится к шагу **1.8** и пересекается с **9.1.2**. Тип: **логика**, **backend**.

### Средние

- **`apps/webapp/src/shared/ui/AskQuestionFAB.tsx`** — логика Mini App и анти-мигание есть; остаются инлайн-стили у оболочки и класс `ask-question-fab`. Шаг **1.3**. Тип: **фронт**, **стили**.
- **`apps/webapp/src/shared/ui/AppShell.tsx`** — `visible={user !== null}` без `channelContext`, если продукт требовал комбинацию. Шаг **1.3**. Тип: **архитектура**, **фронт**.
- **Интегратор Max** — цепочка привязки не верифицирована в этом аудите. Шаг **1.2**. Тип: **логика**, **backend** (вне webapp).
- **Сессия доктора** — `apps/webapp/src/modules/auth/service.ts`: поведение «в течение дня» только стенд/ручной прогон. Шаг **1.6**. Тип: **логика**.
- **CMS** — сохранение контента не ревьюилось построчно. Шаг **1.7**. Тип: **логика**.

### Мелкие

- **`apps/webapp/src/app/app/patient/profile/ProfileForm.tsx`** — для непривязанного телефона остаётся `Link` на `/app/patient/bind-phone` (для первичной привязки может быть ок). Шаг **1.1**. Тип: **фронт**.
