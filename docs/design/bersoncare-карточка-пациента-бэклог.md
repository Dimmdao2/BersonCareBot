## 1. Обзор

Роль: «что с человеком сейчас и что от меня требуется» за пять секунд. Колонки 50/50, таб по умолчанию.

Осталось (реализация):

- **Левая (клиника и реабилитация):**
  - KPI-строка: «Контроль — через N дн» (нужна длительность текущего этапа в модели программы) и «Абонемент — осталось n из N» (если активен).
  - **Сигналы** — тот же источник, что «Сигналы пациентов» на «Сегодня», отфильтрован по пациенту (не выполняет упражнения N дней, тест ждёт проверки). Виден только при наличии.
  - **Актуальные симптомы** — основной ⚑ крупно с бейджем N/10 и «было X/10», остальные мелкими строками. Зависит от модели жалоб. Диагноз сюда НЕ выносим.
  - **Динамика симптомов** — крупный график 0–10 по визитам, линия на каждый актуальный симптом. Зависит от `complaint_update` со шкалой.
  - **Выполнение упражнений** — календарь месяц/неделя, заливка дня по % выполненных, сегодня — жёлтая обводка, сводка «N из M дней · X%». Пока по составу программы.
- **Правая (порядок зафиксирован):** Заметки (📌 закреплённые сверху) → Задачи (по дедлайну, общие с «Сегодня») → Программа и комментарии (пейджер этапов, блоки, строки упражнений с иконкой-гирей последней отметки, 💬 счётчик, красная точка непрочитанных) → Сообщения (только контекст, без поля ответа).

Идеи / следующие этапы (из канона):

- Самоотчёты пациента по симптомам лягут в «Динамику» и **заменят «Самочувствие»** — отдельного блока самочувствия на Обзоре нет.
- Раскрытие упражнения с относительной динамикой (повторы · сложность · вес, без абсолютных цифр) — позже.
- Расписание по дням/блокам для календаря выполнения — позже (сейчас по составу программы).

## 2. Карта — клиническое ядро (самый большой кусок)

Модель данных и UI зафиксированы в каноне. Это новый таб, тут основная инженерная работа.

Осталось (реализация):

- **БД-модель (сущности сразу, справочники потом):**
  - `visit` — дата, локация, тип (первичный/повторный), длительность, услуга, опциональная связь с booking. Текстовые секции (осмотр, манипуляции, результаты пробного лечения, рекомендации) — свободный текст.
  - `complaint` — свободный текст, флаг приоритета ⚑, статус (актуальна/снята), ссылка на визит-источник.
  - `complaint_update` — текст динамики + **выраженность 0–10** + ссылки на жалобу и визит. На повторном визите пишется к каждой актуальной жалобе, тут же можно снять.
  - `diagnosis` + `diagnosis_update` — статус (актуальный→уточнён→снят), даты, флаг, ссылка на визит.
  - `diagnosis_catalog` — **собственный справочник клинических диагнозов** (не МКБ, не свободный текст). Автокомплит в визите + «+ Создать в справочнике», переиспользуется между пациентами.
  - `file` — категория (выписка/снимок/анализ/фото теста/прочее), дата, опциональная привязка к визиту.
  - «Актуальное состояние» в карте = **проекция** этих записей, не копия.
- **Форма визита:**
  - Первичный: «Жалобы» и «Предв. диагноз» — список строк-сущностей (⚑ + текст + ✕, Enter = новая строка), не textarea.
  - Повторный: «Динамика симптомов» — карточки по жалобам (текст жалобы серым + поле динамики + «было N/10 → [новое]» + чекбокс «решена — снять»); «Уточнение диагноза» — карточки по диагнозам + плюс для новых.
- **График динамики симптома** 0–10 по визитам; мини-спарклайны в строках жалоб; бейдж текущего значения у актуальной жалобы.
- **Механика «+ Новый визит»:** карта блюрится, история визитов прижимается к форме, панель формы 45%, независимый скролл истории и формы, ◀/▶ сворачивание истории, ✕ закрывает. Дата/локация/услуга/длительность подставляются из записи на сегодня.
- **Анамнез:** таблицы (Травмы и операции · Болезни, стрессы · Образ жизни) + датированные записи, правка по клику, добавление плюсиком у заголовка.

Отложено на этап D (после обкатки практикой): справочники зон/сторон/характеров боли, структурные поля внутри жалобы. Добавятся рядом с текстовым телом без миграции.

## 3. Программа

Правая колонка (этапы/группы/упражнения) — рабочая, **не трогаем** (требование Дима). Левая колонка переработана в этой сессии (каркас _26_4).

Осталось:

- Реализовать переработанную левую колонку: компактный статус программы (деструктив в меню ⋯), **«Лента пациента»** в главном блоке (отметки + заметки пациента, ответ отсюда, неотвеченные подсвечены), «Заметки специалиста» со свёрнутым полем ввода, «Общие рекомендации» вместо «этап 0», «История правок» свёрнута вниз.
- Решить именования (открыто): «Журнал выполнения» → «Лента пациента»? Разнести ли «Заметки специалиста» и обратную связь пациента, или единая переписка по программе.
- **Мост «Оформить визит»:** из Записей (на прошедшей записи) — переход в форму визита в Карте.
- Точечные правки правой колонки — позже, по живому процессу.

## 6. Учётка

Проработана в этой сессии (каркас _26_4): Личные данные, Контакты и каналы, Сопровождение, Платежи и расчёты, Репутация записи (переехала из Записей), Блокировки и доступ, Администрирование (merge + audit + технические ID).

Осталось / решить:

- **«Платежи» — решить источник данных.** Деньги ведём здесь (ручной ввод или интеграция оплат) или во внешней бухгалтерии? Без решения блок мёртвый (ручные блоки в реальности не заполняются).
- **«Сопровождение» ужать** — наполовину дублирует Обзор (программа/этап/абонемент уже там). Оставить чисто административное: lifecycle-статус, даты, источник.
- **merge** — связать с существующим «Мердж пациентов».
- **«Отображаемое имя / скрытое ФИО»** — Дим хочет пересмотреть концепцию по всей системе. Отдельная задача, не только в Учётке. Пока слово «скрытое» из ярлыка убрано.
- **audit** — определить набор: только ключевые системные события или полный журнал правок полей.

## (Убрано) Аналитика

Решение этой сессии: per-пациентную аналитику-вкладку с графиками сейчас не делаем. Для работы пациента нужны Обзор, Записи, Учётка, Программа; детальные графики по одному пациенту — хотелка, дублируют Обзор (снимок) и общую «Аналитику → Использование» (агрегат).

Не потерять: два реально новых сигнала — **«давно не приходил»** и **«открывает, но не отмечает»** — вынести в Обзор / «Сегодня» как алерты, а не как отдельный дашборд.

## Сквозные задачи (касаются всей карточки)

- **Шапка карты:** ФИО + ★ «На сопровождении · N мес»; под ФИО — ДР · возраст; телефон строкой (копируемый) + иконки каналов (чат / Telegram / MAX / Email, активны подключённые). Справа мини-сводка: Прошлый визит (дата · тип · город) · Следующая запись · Визитов всего (клик «детали ▾» — отмены, переносы).
- **Поиск пациентов по всем полям одновременно** (имя/фамилия, в т.ч. скрытые; отображаемое имя; телефон; email; Telegram; MAX) — бэкенд доработать. Это на странице Пациенты, но это вход в карту.
- **Терминология «пациент» / «клиент»** — финализировать единообразно (флаг в Настройках).
- **«Открыть карту пациента»** — единый синий CTA из превью, записей, чатов, «Сегодня».
