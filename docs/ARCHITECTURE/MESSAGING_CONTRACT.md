# Контракт отправки сообщений (message intents)

Описывает формат исходящих намерений `message.send` и `message.edit`, а также поддержку разметки текста (жирный, ссылки в тексте).

## Типы интентов

- **message.send** — отправить новое сообщение.
- **message.edit** — изменить текст (и при необходимости клавиатуру) существующего сообщения.
- **message.replyMarkup.edit** — изменить только клавиатуру сообщения.

## Payload сообщения

Общие поля payload для `message.send` и `message.edit`:

| Поле | Тип | Описание |
|------|-----|----------|
| `recipient.chatId` | number | Идентификатор чата (например, Telegram chat_id). |
| `message.text` | string | Текст сообщения. Может содержать разметку, если задан `parse_mode`. |
| `replyMarkup` | object | Опционально. Клавиатура (reply или inline) в формате канала. |
| **parse_mode** | `'HTML' \| 'Markdown'` | Опционально. Режим разметки текста: жирный, ссылки в тексте и т.д. |

Для `message.edit` дополнительно:

| Поле | Тип | Описание |
|------|-----|----------|
| `messageId` | number | Идентификатор редактируемого сообщения. |

## Единый формат разметки: HTML

В контракте и в шаблонах контента используется **HTML** как общий формат разметки:

- **Жирный:** `<b>текст</b>`
- **Ссылка в тексте (без показа URL):** `<a href="https://example.com">подпись</a>`
- При необходимости: `<i>...</i>`, `<code>...</code>`

В шагах сценариев (scripts) для сообщений с такой разметкой задаётся параметр **parseMode: "HTML"**; в payload интента он передаётся как **parse_mode: "HTML"**.

### Поддержка по каналам

- **Telegram** — нативная поддержка `parse_mode: 'HTML'` в Bot API.
- **Макс / VK** — при добавлении адаптеров: либо передача того же HTML в API канала (если поддерживается), либо конвертация в формат канала в адаптере. Контент и контракт остаются в HTML.

## Ссылки в списке записей (Мои записи)

В сценарии «Мои записи» каждая запись может отображаться как **ссылка в тексте сообщения** (кликабельный текст без показа полного URL).

- Шаблон `bookingsList` использует разметку вида  
  `<a href="{{item.link}}">{{item.recordAt}} — {{item.status}}</a>`  
  при наличии `item.link`.
- Поле **link** для элемента списка записей:
  - берётся из payload RubiTime (`link`, `url` или `record_url` в `payload_json`);
  - если не задано — подставляется fallback: ссылка на кабинет пациента в webapp (`APP_BASE_URL/app/patient/cabinet`), если в конфигурации задан `APP_BASE_URL`.

Таким образом, у каждой записи в Telegram отображается кликабельная строка «дата — статус», ведущая либо на страницу записи (если есть link от RubiTime), либо в кабинет.
