1. Финальная концепция платформы

Общая модель

Платформа состоит из трех слоев:

1. Messaging layer
	•	Telegram
	•	VK
	•	Max
	•	другие каналы позже

Используется для:
	•	уведомлений
	•	напоминаний
	•	быстрых действий
	•	перехода в Web-App
	•	записи на прием
	•	вопросов пользователю

⸻

2. Web-App (основной интерфейс)

Главный пользовательский интерфейс системы.

Через Web-App пользователь:
	•	смотрит уроки
	•	выполняет программы
	•	ведет дневники
	•	смотрит статистику
	•	управляет напоминаниями
	•	работает со своим кабинетом

⸻

3. Backend платформа (BersonCareBot)

Серверная система:
	•	управление пользователями
	•	управление событиями
	•	рассылки
	•	напоминания
	•	интеграции
	•	программы
	•	дневники

⸻

2. Основная логика взаимодействия

2.1 Основной вход пользователя

Пользователь взаимодействует через:

Telegram / другой мессенджер

Кнопки: Записаться, Меню (и др.). Любое сообщение пользователя пересылается админу. Кнопка Меню открывает Web-App (при переходе по ссылке).

⸻

3. Web-App

3.1 Общая структура

Верхняя панель:
	•	название приложения
	•	справа кнопка настройки

Настройки содержат:
	•	профиль
	•	подписки
	•	платежная информация
	•	управление уведомлениями

⸻

3.2 Главный экран (меню)

Основное меню Web-App.

Пункты:

1️⃣ Мои покупки
2️⃣ Скорая помощь
3️⃣ Полезные уроки
4️⃣ Персональный помощник
5️⃣ Кабинет клиента

Порядок может корректироваться.

⸻

4. Раздел «Скорая помощь»

Отдельный раздел.

Предназначен для ситуаций острой боли.

Примеры подразделов:
	•	острая боль в спине
	•	боль в шее
	•	головная боль
	•	паническая атака
	•	проблемы с дыханием

Содержимое:
	•	рекомендации
	•	упражнения
	•	видео
	•	инструкции

⸻

5. Полезные уроки

Раздел образовательного контента.

Содержит:
	•	уроки
	•	упражнения
	•	тренировки
	•	обучающие материалы

⸻

6. Уроки и упражнения

Видео открываются внутри Web-App.

Пример сценария:

Бот присылает:

Пора сделать разминку для шеи

Кнопки:
	•	Сделал
	•	Напомнить позже
	•	Посмотреть упражнение
	•	Пропускаю (зато честно)

Если нажать посмотреть упражнение:

→ открывается Web-App
→ конкретное видео.

⸻

7. Механика напоминаний

Напомнить позже

При нажатии:

Показываются варианты:
	•	через 30 минут
	•	через 1 час
	•	через 2 часа

И возможность:

Написать свое время (формат HH:MM)

⸻

8. Дневники

8.1 Общая архитектура

Все дневники используют единую модель данных.

Тип записи:
	•	симптом
	•	стресс
	•	боль
	•	другие состояния

Разница между ними — тип записи.

⸻

9. Дневник симптомов

Добавление через мессенджер.

Сценарий:

Бот спрашивает:

На сколько баллов сегодня болит спина?

Пользователь пишет:

5

Ответ бота:

Принято

Кнопки:
	•	Указать характер боли
	•	Добавить комментарий

Оба действия опциональны.

⸻

10. Дневник стресса

Отдельный раздел UI.

Но та же модель данных.

Содержит:
	•	уровень стресса
	•	комментарий
	•	причина стресса

Дополнительно:
	•	медитации
	•	антистресс упражнения
	•	тесты

⸻
