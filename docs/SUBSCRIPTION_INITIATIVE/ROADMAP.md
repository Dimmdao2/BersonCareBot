## ST-01 — Сервис-операция bulk-пересчёта (ядро боли #2) `backend` `complex`
**Цель:** одна идемпотентная операция «Пересчитать»: для активного абонемента пациента найти все прошедшие записи (appointments) по услугам абонемента в окне `[soldAt; now]`, ещё не списанные, и списать их (`consume` в `be_package_usages`), уменьшив остаток. Не списывать дважды; не уходить в минус (стоп при исчерпании баланса позиции).
**Файлы (scope):** `apps/webapp/src/modules/memberships/service.ts`, `ports.ts`, `types.ts` (+ новый unit-тест-файл рядом). Возможно `infra/repos/pgMemberships.ts` (если нужен новый запрос выборки прошедших несписанных записей — на базе существующего `listPackageAppointmentSessionSources`).
**Чек-лист:**
- [ ] Новый метод сервиса (напр. `recalcPastSessionsForPackage`) — переиспользует `listPackageAppointmentSessionSources` / `listPatientPackageSessions(includePast:true)` для выборки, НЕ дублирует запрос.
- [ ] Окно: запись `startsAt >= soldAt` и `startsAt < now`; статус записи валиден для списания (см. OQ-7 — какие статусы; по умолчанию — состоявшиеся/визит-подтверждённые, не отменённые).
- [ ] Списываем только `linkage === "none"` (ещё не зарезервирована/не списана) — идемпотентность: повторный вызов не списывает повторно.
- [ ] Подбор позиции абонемента по `serviceId` записи (как `findItemForService`); если услуга записи не покрыта абонементом — пропустить (учесть в OQ-6).
- [ ] Стоп при исчерпании остатка позиции; возврат сводки `{ debited, skipped[], outOfBalance[] }`.
- [ ] Каждое списание = `consume` в ledger + `setAppointmentPackageUsageRef(appointmentId, usageId)` (чтобы запись получила пометку для календаря/визита) + `be_package_history_events`.
- [ ] best-effort `refreshPackageCalendarForAppointment` для каждой списанной записи (как в `consumeForAppointment`).
- [ ] Unit-тесты: пустое окно; запись до `soldAt` (не списывать); уже списанная (idempotent); услуга вне абонемента; исчерпание баланса; повторный вызов = no-op.
**DoD:** код + тесты зелёные (scoped, через `run-tests.sh`); трассировка показывает корректные ledger-записи; идемпотентность доказана.
**Зависит от:** —

## ST-02 — API «Пересчитать» (доктор) `backend` `complex`
**Цель:** эндпоинт, вызывающий ST-01, с роль-гейтом и (см. OQ-1) проверкой владения пациентом.
**Файлы:** новый `apps/webapp/src/app/api/doctor/booking-engine/patient-packages/[id]/recalc/route.ts` (+ `.test.ts`). Шаблон — соседний `.../[id]/consume/route.ts`.
**Чек-лист:**

- [ ] Вызов `deps.memberships.recalcPastSessionsForPackage(...)` с `organizationId` из гейта.
- [ ] Возврат сводки списания (для тоста в UI).

**DoDः** route + тест; гейт корректен.
**Зависит от:** ST-01.
