# 📥 INBOX — входящие задачи владельца

Сюда **агенты и владелец складывают новые задачи** (любой степени проработки — от одной строки до черновика инициативы). Оркестратор регулярно сюда смотрит и забирает следующее в работу.

## Как положить задачу
Создай файл `docs/_INBOX/<краткое-имя>.md`. Минимум — заголовок + что нужно (своими словами). Можно сырьём. Опционально: приоритет, ссылки, скрины.
Шаблон — `docs/_INBOX/_TEMPLATE.md`.

## Как оркестратор это потребляет (правило)
1. **Когда текущий пул задач заканчивается** (или на регулярной проверке каждый проход) — оркестратор:
   - читает реестр `docs/INITIATIVES.md` → если есть **⏳ QUEUED** инициатива → берёт следующую по порядку/приоритету → в работу (цикл §4 схемы).
   - если QUEUED пуст → смотрит **`docs/_INBOX/`** → если есть новая задача → запускает **фазу планирования** (схема §3) → задача превращается в инициативу (папка `docs/<NAME>_INITIATIVE/`), статус ⏳ QUEUED/▶️ ACTIVE в реестре, **файл из INBOX перемещается в её папку как `REQUIREMENTS.md` (дословно интент владельца)**.
2. Готовую инициативу → ✅ ARCHIVED (перенос в `docs/_ARCHIVE/`), отметка в реестре.

Канон процесса — `docs/AGENT_AUTORUN_SCHEME.md`. Раскладка статусов — `docs/INITIATIVES.md`.
