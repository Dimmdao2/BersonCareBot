# BersonCare — обзор продукта

**Статус:** канонический product brief (не IA по экранам).  
**Обновлять:** при крупных продуктовых релизах и смене позиционирования.  
**Не дублирует:** детали маршрутов, модалок, API — см. связанные документы в конце.

## Суть

**BersonCare** — платформа сопровождения реабилитации и профилактики для частной практики специалиста. Это не «чат-бот с кнопками», а **основной кабинет в PWA** (пациент и специалист). **Web Push — основной канал уведомлений** в PWA; Telegram, MAX, SMS и email — дополнительные каналы доставки. См. [`ARCHITECTURE/NOTIFICATION_CHANNELS.md`](ARCHITECTURE/NOTIFICATION_CHANNELS.md).

Пользователь может зайти из мессенджера (Mini App), установить PWA или открыть сайт в браузере — ядро сценариев живёт в webapp под `/app`.

---

## Три слоя (как устроен продукт)

| Слой | Роль |
|------|------|
| **Webapp (PWA)** | Главный интерфейс: кабинет пациента, кабинет врача/админа, публичная запись, лендинг |
| **Messaging** | Уведомления (**push-first** в PWA), напоминания, быстрые действия, переход в приложение, часть сценариев записи и вопросов |
| **Backend** | Единая PostgreSQL (`public` + `integrator`), integrator (webhook, доставка, worker, scheduler), media-worker (видео/HLS) |

## Для пациента — что даёт платформа

Ежедневный **помощник** и **место выполнения назначений** специалиста. Любая функция должна отвечать хотя бы на один из вопросов ниже.

### Сегодня и привычки

- Главная **«Сегодня»**: разминка дня, ближайшее напоминание, прогресс, чек-ин самочувствия, блок записи, материалы по ситуации, ссылка на активную программу реабилитации.
- **Напоминания** — личные правила и напоминания от специалиста (упражнения, симптомы, разделы контента, программа реабилитации); доставка в PWA, push и мессенджеры по настройкам.

### Назначения специалиста

- **Программа реабилитации** (план лечения): этапы, упражнения, тесты, рекомендации, видео; выполнение, обсуждение пунктов с врачом, отметки прогресса.
- **Клинические тесты** и результаты в рамках программы.

### Самонаблюдение

- **Дневник** (`/app/patient/diary`): симптомы, занятия, динамика (в т.ч. недельный обзор самочувствия).

### Запись и визиты

- **Запись на приём**: собственный движок записи (`/book/*`, кабинет «Запись»); на переходный период — синхронизация/наследие **Rubitime** (уведомления о записях в мессенджер/SMS).
- Просмотр предстоящих и прошлых визитов, подготовка к приёму (где включено в контент).

### Контент и обучение

- **Образовательные материалы** из CMS: статьи, видео, разделы «ситуаций», справка, скорая помощь — без выноса защищённого видео наружу.
- **Курсы** как отдельная витрина self-paced — в **roadmap**, не ядро текущего MVP (см. § «Реализовано vs в планах»).

### Связь со специалистом

- **Сообщения** (inbox): переписка, рассылки врача, события по записи в одной ленте; push ведёт в чат.
- Вопрос через **мессенджер** (Telegram/MAX) с пересылкой специалисту — для тех, кто не в PWA.

### Аккаунт и доступ

- Вход: телефон, email, OAuth, привязка Telegram/MAX; **PWA** с установкой с лендинга.
- Профиль, темы уведомлений, установка приложения, управление каналами связи.

## Для специалиста и админа — что даёт платформа

**Рабочее место** клинического специалиста и **редактор контента** приложения для пациентов. Не техническая SaaS-админка всей платформы (она задумывается отдельно).

### Рабочий день

- **«Сегодня»**: inbox дня — записи, заявки, непрочитанные сообщения, события по пациентам, тесты к проверке.
- **Календарь** и операции с записями (собственный движок + интеграции).

### Работа с пациентом

- **Список клиентов** и **карточка пациента**: контакты, каналы, заметки, программы, дневник, переписка, история записей и коммуникаций.
- **Назначение и ведение программы реабилитации** из шаблонов; правки инстанса; обсуждение по пунктам программы (PWA + ответы из Telegram/MAX).

### Каталог «Назначения»

- Библиотека для сборки назначений: **упражнения**, **комплексы ЛФК** (каталог врача), **клинические тесты** и наборы, **рекомендации**, **шаблоны программ** лечения.
- Каталоги — **инструмент назначения**, точка входа из карточки пациента, а не самоцель.

### Контент для пациентского приложения

- **CMS**: разделы и страницы, главная пациента (блоки «Сегодня»), медиатека, справка `/help`, мотивации, материалы по подписке/ситуациям.
- Настройки отображения, режимы (техработы patient app, тестовые аккаунты), интеграции в **Settings** (`/app/settings`, `/app/doctor/admin/*`).

### Коммуникации

- **Единый чат** с пациентами; **массовые рассылки** с превью и журналом доставки (PWA-чат + push + HTML в мессенджерах).
- Операторские метрики напоминаний и доставки.

### Операции и аналитика

- **System health**, журнал аудита, продуктовая аналитика, статистика материалов и напоминаний.
- Управление записью: услуги, ресурсы, оплаты (этапы own booking engine), Rubitime как мост на переходный период.

## Каналы доставки

| Канал | Типичное использование |
|-------|------------------------|
| **PWA / браузер** | Основной UI, чат, программы, дневник, запись |
| **Telegram** | Mini App, уведомления о записях, меню, ответы врача, напоминания |
| **MAX** | Аналог Telegram для части аудитории |
| **SMS** | Fallback при отсутствии привязки мессенджера (запись, OTP) |
| **Web Push** | Напоминания и события в установленной PWA (контур развивается; ключи VAPID — в admin Settings) |

## Реализовано vs в планах

| Область | Сейчас | Дальше / отложено |
|---------|--------|-------------------|
| PWA + кабинеты patient/doctor | ✅ Ядро в production | Полировка IA — [`APP_RESTRUCTURE_INITIATIVE/ROADMAP_2.md`](APP_RESTRUCTURE_INITIATIVE/ROADMAP_2.md) |
| Программы реабилитации, дневник, напоминания | ✅ | Proactive-сигналы для врача — backlog |
| Собственный движок записи | ✅ Этапы 1–9 закрыты | SaaS-мультитенант, полный отказ от Rubitime — по [`OWN_BOOKING_ENGINE_INITIATIVE/README.md`](_ARCHIVE/OWN_BOOKING_ENGINE_INITIATIVE/README.md) |
| Rubitime | ⚙️ Мост + legacy-уведомления | Стабилизация перехода завершена; канон — own booking |
| Курсы (отдельная сущность) | ⏸ Частично через контент/программы | [`COURSES_INITIATIVE/README.md`](COURSES_INITIATIVE/README.md) — **отложено** |
| Web Push (полный контур) | ⚙️ VAPID в админке, tick напоминаний | Подписки/SW/backlog — [`PWA_INITIATIVE/BACKLOG.md`](_ARCHIVE/PWA_INITIATIVE/BACKLOG.md) |
| Онлайн-консультации реабилитация/нутрициология вне Rubitime | 📋 ТЗ | [`ONLINE_CONSULT_REHAB_NUTRITION_BOOKING_INITIATIVE/README.md`](ONLINE_CONSULT_REHAB_NUTRITION_BOOKING_INITIATIVE/README.md) |

Условные обозначения: ✅ — основной сценарий есть; ⚙️ — работает с ограничениями или в переходе; ⏸/📋 — сознательно не в ближайшем фокусе.
