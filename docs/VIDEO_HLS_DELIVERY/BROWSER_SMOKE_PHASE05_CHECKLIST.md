# Phase 05 — browser smoke (patient HLS / MP4)

**Назначение:** однократная или перед релизом приёмка **реального** воспроизведения в браузере (аудит phase-05 Major).  
**Не хранить:** пароли, полные presigned URL, секреты bucket.

## Предусловия

- `video_playback_api_enabled = true` в admin `system_settings` (тестовое окружение).
- Сессия пациента (тот же пользователь, что смотрит `/app/patient/content/...`).
- Тестовая страница контента: видео из библиотеки (`video_type=api`) с **готовым HLS** (`delivery: hls` в ответе playback).
- Контрольная страница: тот же тип контента при **выключенном** флаге или медиа **без** HLS-ready — ожидается только MP4 (`/api/media/{id}`).

## Чеклист

| # | Браузер | Сценарий | Ожидание | Pass/Fail |
