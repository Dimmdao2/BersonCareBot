# RISKS — test optimization track

1. **Ложное удаление workflow-тестов** — `e2e/*-inprocess.test.ts` могут проверять цепочку из нескольких роутов/сессий; визуально «дублируют» один `route.test.ts`, но покрывают другой сценарий.
2. **Флейки, замаскированные под дубли** — нестабильный тест могут «оптимизировать» удалением вместо исправления.
3. **Изменение CI semantics** — исключение файлов из `vitest` `include` или отдельного job меняет гарантии merge; любое такое изменение должно быть явным и задокументированным.
4. **Случайный выигрыш по времени** — один прогон не доказывает устойчивость; нужны повторы или сравнение в CI.
5. **Потеря security/contract assertions** — статусы 401/403, подпись integrator, идемпотентность, форма JSON ответа; удаление теста без переноса assertion → регрессия без падения CI (если никто не ловит покрытие).
