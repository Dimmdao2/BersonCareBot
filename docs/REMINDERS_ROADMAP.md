# Roadmap: пользовательские напоминания и контентные разделы

## Фаза 1 — Базовые напоминания

- Доменная модель: правила (категория, расписание, пресеты), occurrences, статусы доставки.
- Хранение: `user_reminder_rules`, `user_reminder_occurrences`, `user_reminder_delivery_logs`.
- Действия: `reminders.rules.get`, `reminders.rule.toggle`, `reminders.rule.cyclePreset`, `reminders.dispatchDue`.
- Telegram: раздел «Персональные напоминания» в меню, включение/выключение по категориям, смена пресета расписания.
- Scheduler: по `schedule.tick` выполняется `reminders.dispatchDue` (отправка просроченных напоминаний).

## Фаза 2 — Напоминания с контентом

- Связка напоминания с материалом: при отправке напоминания добавлять ссылку на раздел/урок (например, из контент-сервиса).
- Порт `ContentCatalogPort.getRandomItem` для выбора материала по разделу/категории.

## Фаза 3 — Защищённый доступ к контенту

- Временные персональные ссылки: `ProtectedAccessPort.issueAccess`, сохранение грантов в `content_access_grants`.
- Подпись/токен (HMAC) для доступа к контенту по срокам действия.

## Фаза 4 — Контентные витрины

- «Полезные уроки» и «Скорая помощь»: кнопки в меню, открытие раздела через `ContentCatalogPort.getSectionLink` (или защищённую ссылку в будущем).
- Расширение каталога разделов и интеграция с внешним контент-сервисом.
