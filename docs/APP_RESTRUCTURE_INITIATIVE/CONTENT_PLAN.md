# CONTENT_PLAN — главная пациента (материалы и блоки)

Рабочий контент-план для **наполнения CMS и блоков главной** (`/app/patient`) в рамках [`APP_RESTRUCTURE_INITIATIVE/README.md`](README.md) / [`TARGET_STRUCTURE_PATIENT.md`](TARGET_STRUCTURE_PATIENT.md). Источник формулировок и таблиц — архивная инициатива [`PATIENT_HOME_REDESIGN_INITIATIVE`](../archive/2026-05-initiatives/PATIENT_HOME_REDESIGN_INITIATIVE/README.md) (Phase 0); данные перенесены без изменения смысла.

**Статус:** черновик для владельца контента и админки; строки таблиц пополняются по факту продакшена.

## Инварианты (не нарушать)

Этот файл **не является источником хардкода**. Runtime главной — БД:

- `content_sections` — разделы, `icon_image_url`, `cover_image_url`, таксономия (`kind` / `system_parent_code` — см. [`CMS_RESTRUCTURE_PLAN.md`](CMS_RESTRUCTURE_PLAN.md));
- `patient_home_blocks` — видимость и порядок блоков;
- `patient_home_block_items` — что показывается в блоке (`target_type`, `target_ref`).

**Slug-и в таблицах ниже — редакционные рекомендации для CMS**, не контракт приложения:

- не переносить их в React, сервисы, API, seed как обязательные значения;
- не делать `switch` по slug из этого файла;
- в тестах — только нейтральные fixture-slug, если нужна проверка generic-поведения.

Видео / иконки / обложки — через CMS-медиатеку и поля разделов / настройку блоков на **`/app/doctor/patient-home`**.

## E. CMS-гайд (материалы и обложки главной)

Правила для полей и ассетов (техсетка UI — в [`patient-home.md`](../../apps/webapp/src/modules/patient-home/patient-home.md) и в журнале архивного редизайна [`LOG.md`](../archive/2026-05-initiatives/PATIENT_HOME_REDESIGN_INITIATIVE/LOG.md)).

### `useful_post.badge_label`

- Рекомендация: **≤ 12 символов**, верхний регистр.
- Дефолт в продукте: «Новый пост».
- На узких экранах (≤ 360px) бейдж ограничен по ширине с эллипсисом — длинные надписи обрезаются с `…`.

### Hero `daily_warmup` — изображение

- Слот на главной: mobile порядка **132×156 px**, desktop (lg+) — ориентир в `patientHomeHeroImageSlotClass` в коде (~224×262 px).
- `object-contain`, важное содержимое — **справа внизу** слота.
- В CMS готовить crop ближе к **3:4** или **4:3**.

### `useful_post` — обложка

- Full-bleed, `object-cover object-center`; градиент снизу для читаемости заголовка. Проверять контраст на ярких фото.

### Hero «Разминка дня»

- Замена/обрезка файла в CMS — контент-задача; см. слот и crop выше.
