# PHASE G — Тесты и документация

Канон: [`MASTER_PLAN.md`](MASTER_PLAN.md) §5 фаза G, §7–10, `.cursor/rules/pre-push-ci.mdc`.

## 1. Цель этапа

Закрепить качество всей инициативы: **автотесты** на критические инварианты (dedup, open→resolved, recovery once), **документация** для операторов и разработчиков без секретов, **журнал** `LOG.md` с итогами фаз.

## 2. Зависимости

- Выполняется после существенной готовности A–F (можно частично параллелить тесты с F).

## 3. In scope / out of scope

### In scope

- Матрица тестов:
  - unit: dedup key builder, `openOrTouch`, `resolve`, recovery idempotency;
  - nock: каждый probe (если не покрыт в фазе B);
  - integration: admin API + session guard (мок сессии как в существующих `route.test.ts`);
  - опционально e2e smoke вне CI если проект не использует e2e для admin.
- Документы:
  - `docs/ARCHITECTURE/` — короткий runbook: где cron, какие ключи `system_settings`, какие пороги (после подтверждения на хосте для systemd);
  - обновить [`README.md`](README.md) инициативы со статусом «выполнено» при закрытии.
- `LOG.md`: для каждой фазы A–G — дата, что сделано, команды проверки, сознательные отказа.

### Out of scope

- Полный переписанный runbook всего деплоя.
- Нагрузочное тестирование вебхуков.

## 4. Разрешённые области правок

| Область | Пути |
|---------|------|
| Тесты | рядом с изменёнными модулями `*.test.ts`, `*.test.tsx` |
| Доки | `docs/ARCHITECTURE/*`, `docs/OPERATOR_HEALTH_ALERTING_INITIATIVE/*` |
| Не трогать | immutable baseline-доки, если помечены в репозитории |

## 5. Декомпозиция шагов

### Шаг G.1 — Матрица покрытия

**Действия:**

1. Таблица «инвариант → тестовый файл» в `LOG.md` или в этом разделе appendix.
2. Закрыть пробелы.

**Checklist:**

- [ ] `rg reportOperatorIncident` / `openOrTouch` — все объявленные call sites имеют тест или `# reason` в LOG.

**Критерий закрытия:** нет критичных непокрытых веток без явного defer.

---

### Шаг G.2 — Локальные проверки по пакетам

**Действия:**

1. `pnpm lint` / `pnpm typecheck` / целевые `pnpm test` для `apps/webapp` и `apps/integrator` по затронутым областям.

**Checklist:**

- [ ] Нет ошибок в изменённых файлах из `read_lints` IDE.

**Критерий закрытия:** зелёно локально по области.

---

### Шаг G.3 — Полный CI перед merge

**Действия:**

1. По правилу репозитория: `pnpm install --frozen-lockfile && pnpm run ci` перед пушем итоговой ветки.

**Checklist:**

- [ ] См. `.cursor/rules/pre-push-ci.mdc`.

**Критерий закрытия:** CI зелёный.

---

### Шаг G.4 — Документация

**Действия:**

1. Архитектурная заметка: диаграмма потока «probe / webhook → incidents → notify → UI» (mermaid по желанию).
2. Список `error_code` для операторов (краткий глоссарий).

**Checklist:**

- [ ] Нет секретов, токенов, полных URL с query секретов.

**Критерий закрытия:** новый разработчик может включить фичу по доке.

## 6. Definition of Done (фаза G)

- Матрица покрытия заполнена; критические инварианты с тестами.
- `pnpm run ci` зелёный на ветке инициативы.
- `LOG.md` содержит записи по фазам A–G и финальное резюме.
- Архитектурная заметка или раздел в существующем doc добавлен.
