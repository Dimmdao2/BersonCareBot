# Stage 8 - Зачистка, документация, финальная стабилизация

Сводка в общем плане: `MASTER_PLAN.md` -> Stage 8.

## Цель этапа

Убрать устаревшие элементы auth-потока, зафиксировать новую архитектуру в документации и завершить инициативу с прозрачным финальным аудитом.

## Scope (только Stage 8)

- Удаление/деактивация deprecated UI/комментариев.
- Актуализация документации auth и конфигурации.
- Логирование использования SMS (операционный контроль).
- Финальные проверки и подготовка к global final audit/fix.

Не включать:

- Новые продуктовые функции вне ранее согласованных stage.

## Предусловия

- Stages 1-7 имеют рабочие реализации или закрытые rework-пункты.
- `AGENT_EXECUTION_LOG.md` заполнен по этапам.

## Подробный план реализации

### S8.T01 - Очистка legacy UI

1. Удалить/архивировать старый `PhoneInput.tsx`, если больше не используется.
2. Проверить импорт-граф, чтобы не осталось "мертвых" ссылок.
3. Обновить `AGENT_EXECUTION_LOG.md`.

### S8.T02 - Обновление auth документации

1. Обновить `apps/webapp/src/modules/auth/auth.md`.
2. Зафиксировать:
   - Telegram Login как primary в вебе,
   - PIN скрыт в публичном потоке,
   - OAuth backend-only,
   - email как профильная опция.
3. Обновить `AGENT_EXECUTION_LOG.md`.

### S8.T03 - Контракт комментариев

1. В `checkPhoneMethods.ts` и соседних местах убрать устаревшие комментарии.
2. Зафиксировать актуальный контракт: OAuth реализован, но UI-непубличный.
3. Обновить `AGENT_EXECUTION_LOG.md`.

### S8.T04 - SMS logging

1. Добавить/проверить операционный лог факта использования SMS отправки.
2. Лог должен быть пригоден для мониторинга затрат.
3. Проверить, что в логах нет секретов.
4. Обновить `AGENT_EXECUTION_LOG.md`.

### S8.T05 - Док конфигурации

1. Обновить `docs/ARCHITECTURE/CONFIGURATION_ENV_VS_DATABASE.md` под текущие auth keys.
2. Подтвердить, что новые интеграционные ключи хранятся в `system_settings`.
3. Обновить `AGENT_EXECUTION_LOG.md`.

### S8.T06 - Deploy docs cross-check

1. Проверить `deploy/HOST_DEPLOY_README.md` на предмет auth-конфигов.
2. Если есть изменения runtime-фактов - обновить документ.
3. Не добавлять секреты.
4. Обновить `AGENT_EXECUTION_LOG.md`.

### S8.T07 - Финальные проверки этапа

1. Прогнать релевантные тесты auth + `pnpm run ci`.
2. Сверить definition-of-done из `MASTER_PLAN.md`.
3. Подготовить список residual risks для final audit.
4. Обновить `AGENT_EXECUTION_LOG.md`.

## Gate (критерий готовности)

- Документация и код соответствуют новой auth-архитектуре.
- Legacy элементы очищены.
- Операционные логи SMS присутствуют.
- `pnpm run ci` зеленый.

## Артефакты этапа

- `docs/AUTH_RESTRUCTURE/AUDIT_STAGE_8.md` (после AUDIT).
- Обновления в `docs/AUTH_RESTRUCTURE/AGENT_EXECUTION_LOG.md`.
