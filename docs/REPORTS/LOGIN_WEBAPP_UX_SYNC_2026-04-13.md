# Отчёт: вход в webapp, поддержка, согласование с кодом (2026-04-13)

## Контекст

Серия правок в `AuthFlowV2`, `SupportContactLink`, публичный конфиг входа и админские подписи; последующий аудит и доведение до согласованного состояния с документацией.

## Что сделано по результатам аудита

1. **Apple Sign-In**  
   На основном экране входа Apple **не показывается**, если включены Яндекс или Google (продуктовое правило: основной набор — Яндекс, Google, Telegram, Max).  
   Если в админке настроен **только Apple** (Яндекс и Google выключены), кнопка **«Войти через Apple»** снова отображается на `oauth_first`, `landing` и в компактных блоках на шаге телефона / экранах OTP-тупиков — чтобы не ломать такие деплои.

2. **Копирайт шага телефона**  
   Убрано вводящее в заблуждение перечисление Telegram в одном ряду с кнопками OAuth/Max: подпись к компактному ряду строится из фактически показанных способов (**Яндекс, Google, Apple при fallback, Max**); вход через Telegram остаётся **отдельной ссылкой ниже**.

3. **Дублирование UI Max**  
   Вынесен общий компонент **`MaxLoginCta`** (primary / outline) в `AuthFlowV2.tsx` — один источник правды для скелетона и ссылки на бота Max.

4. **VK**  
   Поведение без изменений: `vk_web_login_url` по-прежнему в ответе `GET /api/auth/login/alternatives-config`; кнопка на экране входа будет добавлена при подключении в UI. Тексты в админке (`AuthProvidersSection`) уточняют это.

5. **SupportContactLink**  
   Внутренние пути `/app/…` остаются на **нативном `<a href>`** (полная загрузка документа) для снижения риска «Failed to load chunk» после деплоя. Зафиксировано в `apps/webapp/src/modules/auth/auth.md`.

## Документы, приведённые в соответствие с кодом

- `apps/webapp/src/modules/auth/auth.md` — разделы публичного входа, UI `AuthFlowV2`, поддержка (`SupportContactLink`).
- `docs/README.md` — ссылка на этот отчёт (см. оглавление отчётов / инициативы при необходимости навигации).

## Замечания по-прежнему актуальные

- Дополнительный **HTTP-запрос** `alternatives-config` на каждую загрузку веб-входа (данные для Max и дублирование `telegramBotUsername`); при желании оптимизации — объединить с другими публичными конфигами в один round-trip (отдельная задача).
- **Полная перезагрузка** по внутренней ссылке поддержки — осознанный компромисс против устаревшего клиентского бандла.
