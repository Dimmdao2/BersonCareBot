# План: календарный пояс пациента (IANA)

## Цель

Пациент видит и сохраняет **IANA-зону** для «своих суток» в продукте; при первом входе зона **не остаётся пустой**, если браузер отдаёт `Intl`; подписи в селекторе **читаемы** (релокация, РФ) и **не плодят ложные дубли** из-за ограничений UI-библиотеки.

## Ограничение UI (`react-timezone-select`)

Библиотека **склеивает** опции с одинаковой парой **`(offset, hasDst)`** и оставляет **первую** после сортировки; ключи, которых **нет** в дефолтном списке пакета, считаются **custom** и строка **всегда** показывается (см. `apps/webapp/node_modules/react-timezone-select/dist/index.js`, `options` / `customTimezoneKeys`).

Следствия для подписей:

- Подпись на «проигравшем» IANA в группе **не видна**, но её текст участвует в **`searchTerms`** у выжившей строки — поиск по городу может находить «скрытую» зону.
- Чтобы **город был виден в одной строке**, его нужно переносить на **подпись выжившей** зоны той же группы (как сделано для **+3** / `Europe/Bucharest`, **+4** / `Asia/Dubai`).

2. Нет регрессий в API календарного пояса (тесты + ручной смоук профиля).
