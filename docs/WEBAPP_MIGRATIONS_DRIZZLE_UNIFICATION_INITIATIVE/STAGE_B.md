# STAGE_B — Drizzle Consolidation (Codex)

## Цель

Сконсолидировать webapp schema evolution в Drizzle без регрессий для существующих БД.

## Шаги

1. По результатам Stage A перенести недостающие критичные DDL в Drizzle path.
2. Проверить идемпотентность на уже мигрированной БД.
3. Проверить bootstrap новой/чистой БД, чтобы требуемые колонки/таблицы появлялись без legacy-runner.
4. Зафиксировать совместимость и ограничения (если остаются).
5. Обновить docs инициативы и `LOG.md`.

## Выход этапа

- все runtime-critical изменения webapp покрыты Drizzle;
- явно документировано, какие legacy-миграции остаются и почему.

## Gate закрытия

- целевые миграционные проверки успешны;
- нет блокирующих замечаний по риску повторного DDL;
- в `LOG.md` есть итог этапа B.
