# Script Scenarios Execution Report

Date: 2026-03-07  
Scope: `src/content/telegram/scripts.json`, `src/content/rubitime/scripts.json`

## Grouped problems

1. **Telegram callback/action path все еще частично сломан**
  - `notifications.toggle.*` зависают в состоянии `Loading...`.
  - `bookings.show` зависает без обновления экрана.
  - `booking.open` не дает реакции.

2. **State path сценария вопроса работает не полностью**
  - после экрана `Опишите ваш вопрос` следующий пользовательский текст не приводит к подтверждению и не уходит админу;
  - вероятная проблема в сохранении или повторном чтении `waiting_for_question`.

3. **Rubitime runtime path фактически не подтвержден и выглядит нерабочим**
  - события записи не приводят к доставке сообщений ни пользователю, ни админу.

4. **UX Telegram-меню пока не соответствует ожидаемому**
  - главное меню стало вертикальным / визуально другим;
  - первый `/start` показывает fallback-текст `Выберите действие в меню.` вместо ожидаемого приветствия;
  - на экране уведомлений есть лишняя кнопка `Все категории`.

## Результаты фактической проверки (realtime)

### Подтверждено в бою

- `/start` теперь открывает главное меню.
- Кнопка `Меню` открывает экран с пунктами:
  - `Настройки уведомлений`
  - `Мои записи`
- Экран `Настройки уведомлений` открывается.
- Кнопка `Назад` на экране уведомлений работает.
- Повторный вход в `Меню` работает.

### Работает, но требует правки UX

- главное меню отображается не как раньше: вместо прежнего варианта — вертикально / в другом layout;
- на первый `/start` приходит fallback-текст `Выберите действие в меню.` вместо приветственного сообщения;
- на экране уведомлений отображается лишняя кнопка `Все категории`.

### Не работает

#### Telegram

- `notifications.toggle.*` — по нажатию висит `Loading...`, экран не обновляется;
- `bookings.show` — по нажатию висит `Loading...`, реакции нет;
- `booking.open` — на `Запись на приём` реакции нет;
- `ask.question` — после `Опишите ваш вопрос` отправка текста не приводит ни к подтверждению пользователю, ни к пересылке админу;
- `admin.forward` — админу ничего не приходит.

#### Rubitime

- события записи не приводят к доставке сообщений:
  - ни пользователю,
  - ни админу.

  - отдельно отметить сценарии, которые работают, но требуют UX-правки.

## Classification rule

- **Полностью выполняется** — сценарий не только матчится и парсится, но и проходит текущий runtime-путь без выявленных потерь смысла.
- **Частично / не полностью** — сценарий выбирается и запускается, но в текущем runtime есть выявленные проблемы: пустые кнопки, неотрендеренные `{{...}}`, либо ссылки на поля контекста, которые не заполняются.

## Telegram

### Полностью выполняется

- `telegram.more.menu`  
  Пришло действие `menu.more` -> открывается экран меню с пунктами `Настройки уведомлений` и `Мои записи`.

- `telegram.notifications.show`  
  Пришел callback `notifications.show` -> экран настроек уведомлений открывается.

- `telegram.menu.back`  
  Пришел callback `menu.back` -> возвращает пользователя на предыдущий экран меню.

### Частично / не полностью

- `telegram.start`  
  Пришел текст `/start` -> главное меню открывается.  
  **Проблема:** приходит fallback-текст `Выберите действие в меню.` вместо ожидаемого приветствия, а layout главного меню отличается от прежнего UX.

- `telegram.ask`  
  Пришел текст / действие `question.ask` -> пользователю показывается экран `Опишите ваш вопрос`.  
  **Проблема:** downstream state path не подтвержден; следующий шаг `telegram.ask.question` фактически не отрабатывает.

- главное меню Telegram  
  Базовая навигация работает.  
  **Проблема:** layout кнопок отличается от ожидаемого UX.

- `telegram.ask.question`  
  Пришел обычный текст, пока пользователь находится в состоянии `waiting_for_question` -> должен переслать вопрос админу, вернуть пользователя в `idle`, отправить подтверждение пользователю.  
  **Проблема:** по факту не отрабатывает; вероятно, состояние `waiting_for_question` не сохраняется или не матчится повторно.

- `telegram.booking.open`  
  Пришло действие `booking.open` при `linkedPhone = true` -> должен показать inline-кнопку открытия записи.  
  **Проблема:** по факту реакции нет.

- `telegram.contact.link.request.booking`  
  Пришло действие `booking.open` при `linkedPhone = false` -> перевести пользователя в состояние ожидания контакта и показать клавиатуру с запросом номера телефона.  
  **Проблема:** realtime-подтверждения пока нет; сценарий зависит от неработающего `booking.open` path.

- `telegram.contact.link.request.bookings`  
  Пришел callback `bookings.show` при `linkedPhone = false` -> перевести пользователя в состояние ожидания контакта, показать клавиатуру запроса номера, ответить на callback.  
  **Проблема:** callback path `bookings.show` сейчас зависает в `Loading...`.

- `telegram.contact.link.confirm`  
  В состоянии `await_contact:subscription` пришел контакт / телефон -> привязать телефон, вернуть состояние `idle`, показать главное меню.  
  **Проблема:** realtime-подтверждения пока нет.

- `telegram.contact.link.remind`  
  В состоянии `await_contact:subscription` пришло сообщение без телефона -> повторно показать клавиатуру запроса контакта.  
  **Проблема:** realtime-подтверждения пока нет.

- `telegram.notifications.show`  
  Пришел callback `notifications.show` -> экран настроек уведомлений открывается.  
  **Проблема:** UX не финальный — присутствует лишняя кнопка `Все категории`.

  **Проблема:** по факту зависает в `Loading...`, экран не обновляется.

- `telegram.notifications.toggle.msk`

  **Проблема:** по факту зависает в `Loading...`, экран не обновляется.

- `telegram.notifications.toggle.online`  
  Пришел callback `notifications.toggle.online` -> переключить настройку онлайн-уведомлений, затем перерисовать экран уведомлений и ответить на callback.  
  **Проблема:** по факту зависает в `Loading...`, экран не обновляется.

  Должен отправлять администратору сообщение с вопросом пользователя.

## Final summary

- Все найденные сценарии из Telegram и Rubitime **загружаются и матчятся**.
- В Telegram подтверждено, что базовая навигация ожила: открывается меню, открывается экран уведомлений, работает возврат назад.
- В Telegram при этом остаются серьезные runtime-проблемы в callback/state path: `notifications.toggle.*`, `bookings.show`, `booking.open`, `ask.question`, `admin.forward`.
- UX главного меню и первого `/start` пока не соответствует ожидаемому варианту.
- В Rubitime-сценариях доставка по факту не подтверждена и сейчас выглядит неработающей в runtime.
