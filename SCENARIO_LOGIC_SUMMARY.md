# Логика сценариев (человеческое описание)

## Статус разделения admin/user

- Да, текущие сценарии Telegram разделены:
  - отдельный набор для обычного пользователя;
  - отдельный набор для администратора.
- Это означает:
  - обычный пользователь не попадает в админские сценарии;
  - администратор не попадает в пользовательские сценарии Telegram.

## Telegram — пользовательские сценарии

- Если пользователь пишет `/start`, то бот показывает приветствие и главное меню.
- Если пользователь нажимает «Задать вопрос», то бот просит написать текст вопроса.
- Если пользователь в режиме «ожидаем вопрос» пишет текст, то:
  - бот пересылает вопрос администратору;
  - подтверждает пользователю, что вопрос принят;
  - возвращает пользователя в обычное состояние.
- Если пользователь выбирает «Записаться», и телефон уже привязан, то бот отправляет кнопку для открытия записи.
- Если пользователь выбирает «Записаться», и телефон не привязан, то бот просит отправить контакт.
- Если пользователь нажимает «Мои записи», и телефон не привязан, то бот просит отправить контакт и закрывает callback.
- Если пользователь отправляет контакт в режиме ожидания контакта, то бот привязывает телефон и возвращает главное меню.
- Если пользователь в режиме ожидания контакта пишет не контакт, то бот повторно просит отправить контакт.
- Если пользователь нажимает «Меню», то бот показывает inline-меню с пунктами:
  - настройки уведомлений;
  - мои записи.
- Если пользователь открывает «Настройки уведомлений», то бот показывает текущие переключатели категорий.
- Если пользователь переключает любую категорию уведомлений, то бот сохраняет новое значение и перерисовывает тот же экран.
- Если пользователь нажимает «Мои записи», и телефон привязан:
  - при отсутствии записей бот показывает «активных записей нет»;
  - при наличии записей бот показывает список записей;
  - под списком кнопки «Как подготовиться» и «Адрес и как найти» (друг под другом), внизу «Назад»;
  - по «Как подготовиться» — текст с рекомендациями (выписки, форма одежды);
  - по «Адрес и как найти» — адреса Москва/СПб и ссылка на карту;
  - после этого закрывает callback.
- Если пользователь нажимает «Назад» в inline-меню, то бот возвращает экран меню и закрывает callback.
- Если пользователь в обычном состоянии пишет произвольный текст (не команда/не кнопка), то бот показывает главное меню.

## Telegram — админские сценарии

- Если администратор пишет `/start`, то бот показывает подсказку по админ-меню.
- Если администратор запрашивает статистику по записям, то бот показывает число активных записей.
- Если администратор запрашивает статистику по пользователям, то бот показывает сводку пользователей по интеграциям.

## Rubitime — сценарии

- Если приходит событие о создании записи, то система:
  - сохраняет событие и обновляет состояние записи;
  - отправляет пользователю сообщение «вы записаны на приём».
- Если приходит событие об обновлении записи со статусом `recorded`, то система:
  - сохраняет событие и обновляет запись;
  - отправляет пользователю сообщение о подтверждённой записи.
- Если приходит событие об обновлении записи со статусом `canceled`, то система:
  - сохраняет событие и обновляет запись;
  - отправляет пользователю сообщение об отмене.
- Если приходит событие об обновлении записи со статусом `awaiting_confirmation`, то система:
  - сохраняет событие и обновляет запись;
  - отправляет пользователю сообщение «ожидает подтверждения».
- Если приходит событие об обновлении записи со статусом `moved_awaiting`, то система:
  - сохраняет событие и обновляет запись;
  - отправляет пользователю сообщение «перенос ожидает подтверждения».
- Если приходит событие явной отмены записи, то система:
  - сохраняет событие и обновляет запись как отменённую;
  - отправляет пользователю сообщение об отмене.

## Важное уточнение по Rubitime

- В Rubitime на текущем этапе нет отдельного набора «admin-сценариев».
- Все существующие Rubitime-сценарии относятся к уведомлениям пользователя о состоянии записи.
