# ui

Компоненты интерфейса, общие для страниц приложения.

- **AppShell** — оболочка страницы: контейнер, верхняя панель и область контента. Вариант по умолчанию: заголовок, кнопка «Назад», имя/роль, «Настройки». Вариант `variant="patient"`: шапка в одну линию (стрелка назад | BERSONCARE → меню | гамбургер → боковое меню: профиль, безопасность, уведомления, поддержка, справка). Используется на всех страницах после входа.
- **PatientHeader** — шапка пациента: назад, заголовок BERSONCARE (ссылка на меню), кнопка-гамбургер, открывающая меню справа. Клиентский компонент, подключается через AppShell при `variant="patient"`.
- **AuthBootstrap** — блок входа: при наличии токена в URL — обмен на сессию и редирект; при отсутствии — попытка входа по initData Telegram Mini App. Показывается на `/app` неавторизованному пользователю.
- **FeatureCard** — карточка раздела: заголовок, описание, статус (доступно / скоро / заблокировано), опционально ссылка. Компактный вариант — только заголовок. Используется на главной пациента для пунктов меню.

Внешний вид задаётся классами из `app/globals.css`.
