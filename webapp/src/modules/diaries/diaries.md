# diaries

Дневники пациента: симптомы и ЛФК.

Два подмодуля с общей идеей: хранение делегировано портам (реализации в infra/repos — in-memory или PostgreSQL).

- **Симптомы**: создание отслеживания по названию, список отслеживаний, добавление записей (балл 0–10, тип «в моменте»/«за день»), список записей. Вызывается вебаппом (страница дневника симптомов) и API интегратора (бот при создании трекинга и при запросе списка для кнопок 0–10).
- **ЛФК**: создание комплекса по названию, список комплексов, отметка занятия (сессия), список занятий. Вызывается вебаппом (страница дневника ЛФК, форма «Отметить занятие») и интегратором (бот при создании комплекса и отметке занятия).

Типы и порты описаны в types.ts и ports.ts; сервисы — symptom-service.ts, lfk-service.ts.
