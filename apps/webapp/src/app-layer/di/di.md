# di

Сборка зависимостей приложения (dependency injection).

**buildAppDeps()** возвращает объект с сервисами: auth, users, menu, lessons, emergency, patientCabinet, doctorCabinet, purchases, diaries (symptom + LFK), health, channelPreferences, contentCatalog, media, integrator (события, напоминания). Дополнительно библиотека программ лечения (TREATMENT_PROGRAM_INITIATIVE, фаза 2): **`clinicalTests`** (`createClinicalTestsService`), **`testSets`** (`createTestSetsService`), **`recommendations`** (`createRecommendationsService(recommendationsPort, referencesPort)`) — порты Drizzle из `infra/repos/pgClinicalTests`, `pgTestSets`, `pgRecommendations` при подключённой БД; иначе in-memory реализации для Vitest/`webappReposAreInMemory`. Фаза 3 — **`treatmentProgram`** (`createTreatmentProgramService`): порт `createPgTreatmentProgramPort` / `createInMemoryTreatmentProgramPort`, валидация полиморфных ссылок `createPgTreatmentProgramItemRefValidationPort` / stub для in-memory. Фаза 4 — **`treatmentProgramInstance`** (`createTreatmentProgramInstanceService`): порт `createPgTreatmentProgramInstancePort` / in-memory (в Vitest один **`createInMemoryTreatmentProgramPersistence()`** даёт согласованные `instancePort` + `testAttemptsPort`), снимки блоков `createPgTreatmentProgramItemSnapshotPort` / stub. Фаза 8 — **`courses`** (`createCoursesService`): порт `createPgCoursesPort` / `createInMemoryCoursesPort`; запись пациента на курс вызывает **`assignTemplateToPatient`** у **`treatmentProgramInstance`** (тот же deep copy шаблона). Фаза 6 — **`treatmentProgramProgress`** (`createTreatmentProgramProgressService`): прохождение этапов и тесты; порт попыток/результатов `createPgTreatmentProgramTestAttemptsPort` / in-memory из того же persistence. Фаза 5 — **`comments`** (`createCommentsService`): порт `createPgCommentsPort` / `createInMemoryCommentsPort` (`infra/repos/pgComments`, `inMemoryComments`); единая таблица `comments` (`SYSTEM_LOGIC_SCHEMA.md` §7). Рядом по кабинету врача: **lfkExercises**, **lfkTemplates**, **lfkAssignments** (ЛФК). Хранилища дневников и настроек каналов подключаются из `infra/repos`; при отсутствии БД используются in-memory реализации. **doctorMotivationQuotesEditor** — список `motivational_quotes` для экрана мотивации врача (`pgDoctorMotivationQuotesEditor` / `inMemoryDoctorMotivationQuotesEditor`). Используется на страницах и в API-обработчиках.

Guard для подписанных integrator **GET** маршрутов собран отдельно: `assertIntegratorGetRequest` в `@/app-layer/integrator/assertIntegratorGetRequest` (не часть объекта `buildAppDeps`; вызывается в `route.ts` до `buildAppDeps()`). См. `apps/webapp/src/app/api/api.md` § integrator.

**Import boundary для Route Handlers:** `route.ts` под `app/api` не импортирует `@/infra/*` напрямую — для доступа к бывшим infra-модулям используются фасады в `@/app-layer/**` (логирование, `getPool`, verify подписи, merge/media helpers и т.д.) либо поля объекта `buildAppDeps()`. Перечень и политика: `docs/archive/2026-04-docs-cleanup/test-api-di-optimization/api-di-boundary-normalization/ALLOWLIST_REMAINING_INFRA_ROUTE_IMPORTS.md` и § «API route import-policy» в `docs/ARCHITECTURE/LOW_LEVEL_ARCHITECTURE_AUDIT_AND_REORG.md`.
