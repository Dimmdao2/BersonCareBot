# patient

Страницы личного раздела пациента.

Показываются только пользователям с ролью «пациент». Кнопка «Назад» на подстраницах ведёт в главное меню пациента (`/app/patient`).

- **page.tsx** — главное меню: сетка карточек разделов (дневники, уроки, скорая помощь, кабинет, покупки и т.д.); пункты берутся из модуля меню по роли.
- **cabinet/** — кабинет клиента: описание, ближайшая запись, список предстоящих приёмов.
- **diary/** — дневник: недельный график самочувствия (ComposedChart; TZ календаря пациента; линия только instant). Вкладки симптомов/ЛФК временно не монтируются — см. `diary/diary.md`.
- **lessons/** — полезные уроки: список уроков со ссылками на страницы контента.
- **emergency/** — скорая помощь: список тем со ссылками на контент.
- **content/[slug]/** — одна страница материала (урок или тема) по идентификатору в адресе.
- **purchases/** — мои покупки: курсы, доступы, подписки (пока на мок-данных).

Все страницы используют общую оболочку `AppShell` и при необходимости данные из `buildAppDeps()` (меню, дневники, каталог контента и т.д.).

**Доступ к персональным данным из БД на RSC** (главная — блоки с профилем/каналами/рассылками; кабинет; дневник и журналы; уведомления): перед запросами по `userId` вызывается **`patientRscPersonalDataGate`** в `app-layer/guards/requireRole.ts` (тот же `patientClientBusinessGate`, что у `/api/patient/*`). Подробнее — `docs/ARCHITECTURE/PLATFORM_IDENTITY_SCENARIOS_AND_CODE_MAP.md` §7 и `guards.md`.
