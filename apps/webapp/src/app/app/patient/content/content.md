# content

Раздел контента пациента: страницы материалов по идентификатору в адресе.

- **[slug]/** — одна страница материала (`/app/patient/content/[slug]`). Данные из каталога контента по `slug`; показываются заголовок, картинка, текст, блок «Видео» (плейсхолдер). Открывается из разделов «Полезные уроки» и «Скорая помощь». При отсутствии материала — 404. Текст статьи рендерится через клиентский компонент `MarkdownContent`: для записей с `body_md` — безопасный Markdown (GFM, без raw HTML); для старых записей только с `body_html` — HTML после санитизации (DOMPurify).

Каталог и разрешение по slug — в модуле `content-catalog`.
