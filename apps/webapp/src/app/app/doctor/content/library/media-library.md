# media-library

Экран библиотеки файлов в CMS (`/app/doctor/content/library`).

**Точка загрузки для врача:** полноэкранная библиотека — основной поток для пакетной загрузки, presigned / multipart в MinIO по возможностям клиента (`MediaLibraryClient`). Дополнительно в модалках выбора медиа (`MediaPickerShell` + `MediaPickerPanel`, см. `MediaLibraryPickerDialog` / `MediaLibraryInsertDialog`) есть вкладка **«Загрузить с устройства»**: один файл уходит в `POST /api/media/upload` в папку, согласованную с фильтром пикера (корень или выбранная папка в режиме упражнений). Атрибут `accept` на поле файла согласован с `kind` пикера; после успешной загрузки тип строки (`image` / `video` / …) проверяется перед вызовом `onPick` — несовпадение с полем (например видео в обложку «только картинка») даёт сообщение на русском без подстановки в форму (запись в библиотеке уже создана). В формах медиа по-прежнему не вводится произвольный URL вручную (кроме legacy `https://` в сохранённом контенте/упражнениях).

**Markdown:** вставка из модалки использует [`markdownMediaSnippet.ts`](../../../../../shared/ui/markdown/markdownMediaSnippet.ts) — решение `![alt](url)` vs ссылка по `kind`/`mimeType` и расширению имени файла.

**Поиск в picker (модалка):** при открытии модалки один раз запрашивается `GET /api/admin/media` **без** `q` (лимит до 200). Строка поиска живёт **только во внутренней панели** client-компонента пикера (не в state родительской формы); при вводе список фильтруется **локально** по подстроке в **исходном имени** и **отображаемом названии** (`filterMediaLibraryPickerItemsByQuery` в `useMediaLibraryPickerItems.ts`) — **без** повторных запросов на каждый debounce-цикл. Параллельные запросы при смене источника списка (`kind` / `folderId` → другой `listUrl`) отменяются через **`AbortController`**; устаревшие ответы отбрасываются по монотонному **`requestId`**. Индикатор загрузки списка: **`open && inFlight`**. Родительская форма обновляется **только после выбора** файла. Для режима «картинка или видео» в запросе используется `kind=all`, затем на клиенте список сужается до `image` и `video` (`narrowMediaLibraryPickerItemsByKind`). **Экран библиотеки** (`MediaLibraryClient`) по-прежнему использует серверный `q` и пагинацию — это отдельный сценарий (большой каталог), не путать с picker.

Функции:
- просмотр загруженных файлов;
- редактируемое отображаемое название (без изменения исходного имени файла в хранилище);
- фильтрация по типу;
- сортировка по типу, дате загрузки и размеру;
- метаданные в списке: кто загрузил (ссылка на профиль) и разрешение image/video;
- загрузка новых файлов;
- удаление через меню действий (`...`) с предупреждением о местах использования в CMS.
