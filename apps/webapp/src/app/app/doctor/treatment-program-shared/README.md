# treatment-program-shared

Общие примитивы для экранов врача по программам лечения:

- **`treatmentProgramConstructorShellStyles.ts`** — цвета шапок карточек, классы карточек этапа/«общие рекомендации», `tplToolbarTextBtnClass`; алиасы `TPL_*` совпадают с именами в конструкторе шаблона и указывают на те же строки, что `INSTANCE_*`.
- **`TreatmentProgramLibraryPickerToolbar.tsx`** — поиск + регион + тип нагрузки для picker программы/шаблона (**без** «Без региона» / «Без типа» — только на экранах каталогов врача).
- **`InstanceAddLibraryItemDialog.tsx`** — модалка «Элемент из библиотеки» для экрана назначенной программы; добавление через **`addItemCreate`** (in-memory draft), без немедленного POST. Для **упражнений ЛФК и комплексов ЛФК** — toolbar + `useTreatmentProgramLibraryPickerList`. Разворот набора тестов / комплекса ЛФК использует **`expandLines`** в строках каталога (`buildTreatmentProgramLibraryPickers`). Снимки preview: `treatmentProgramLibraryDraftSnapshot.ts`. Тесты: `InstanceAddLibraryItemDialog.test.tsx`, `treatmentProgramLibraryPickerFilters.test.ts`.
- **`treatmentProgramLibraryTypes.ts`** — типы строк каталога для шаблона и инстанса.
- **`TreatmentProgramDndUi.tsx`** — `@dnd-kit` обёртки: DnD pipeline-этапов и элементов этапа (grip на строке/этапе).
- **`treatmentProgramReorderHelpers.ts`** — расчёт `orderedStageIds` / `orderedItemIds` после DnD и chevrons; `planStageItemDndReorder` для PATCH `groupId` + bulk reorder.
- **`InstanceEditorToolbar.tsx`** / **`InstanceEditorAddStageDialog.tsx`** / **`InstanceEditorStageOrderDialog.tsx`** — sticky шапка редактора **назначенной** программы (три зоны: meta | «Комментарии» → `DoctorProgramInstanceDiscussionDialog` | actions; `INSTANCE_EDITOR_TOOLBAR_STICKY_CLASS` под `DoctorHeader`). Порядок pipeline-этапов — модалка → draft; inline stage DnD на экране инстанса снят. Pipeline-этапы сворачиваются (`Collapsible`); default expanded — `pickDefaultExpandedPipelineStageId` + `useInstanceEditorPipelineStageExpansion` («+ Группа» раскрывает свёрнутый этап). Per-item обсуждение — `DoctorProgramItemDiscussionDialog` + `?discussionItem=`. Instance dialog: `DoctorProgramInstanceDiscussionDialog`, `DoctorProgramDiscussionMessagesPanel`. Timeline: `DoctorProgramInstanceTimelineEventRow` (`program_changed` diff). Legacy `InstanceEditorSaveBar` — `@deprecated`. Тесты: `InstanceEditorToolbar.test.tsx`, `InstanceEditorAddStageDialog.test.tsx`, `InstanceEditorStageOrderDialog.test.tsx`, `instanceEditorDefaultExpandedStageId.test.ts`, `useInstanceEditorPipelineStageExpansion.test.tsx`, `DoctorProgramInstanceDiscussionDialog.test.tsx`, `DoctorProgramDiscussionMessagesPanel.test.tsx`, `TreatmentProgramInstanceDetailClient.phase4.test.tsx`, `TreatmentProgramInstanceDetailClient.phase5.test.tsx`, `TreatmentProgramInstanceDetailClient.phase6.test.tsx`, **`TreatmentProgramInstanceDetailClient.phase7.test.tsx`**.
- **`InstanceEditorUnsavedChangesDialog.tsx`** / **`useInstanceEditorUnsavedGate`** — gate перед сменой статуса этапа/программы при metadata-dirty; кнопки «Сохранить» / «Вернуться к редактированию». Тесты: `InstanceEditorUnsavedChangesDialog.test.tsx`, `doctor-timeline-text.test.ts`, `DoctorProgramInstanceTimelineEventRow.test.tsx`.
- **`instanceEditorDraft.ts`** / **`InstanceEditorDraftContext.tsx`** — in-memory черновик редактора **назначенной** программы. Client id новых строк/групп/этапов — префикс **`draft:`**; metadata и structural для них пишутся в **`itemCreates`** (и поля create), не в `itemPatches` / `itemStructuralPatches` (в т.ч. строки **`lfk_complex_expand`** / **`test_set_expand`** — per-line `groupId`, `status`, `loadSettings`). **`normalizeInstanceEditorDraft`** сбрасывает orphan-патчи на uuid, которых нет в baseline. Сохранение: **`flushInstanceEditorDraft`** → `serializeInstanceEditorBatchDraftForApi` (без `snapshot` в wire) → `POST …/editor-batch`; перед POST — **`instanceEditorLoadSettings`** (диапазоны reps/sets/maxPain); **`saveDraft`** сначала вызывает **`onBaselineSynced`** (refresh с сервера), при stale «не найден» — повторный sync + подсказка в toast. Одно событие `program_changed` (`payload.diff`, timeline). Сервер: **`applyInstanceEditorBatch`** в **`runInMutationTransaction`**; при `itemCreates` / replace — **`buildSnapshot`** (не browser preview). HTTP-ошибки instance routes: **`doctorInstanceRouteErrorStatus.ts`**. **`isFlushableDirty`** — unsaved gate перед status API блокирует только metadata. Разворот комплекса ЛФК: `loadSettings` из шаблона в `expandLines` (`buildTreatmentProgramLibraryPickers`). **Ops:** backfill битых preview-snapshot — [`docs/OPERATIONS/TREATMENT_PROGRAM_EDITOR_DRAFT_SNAPSHOT_BACKFILL.md`](../../../../../../docs/OPERATIONS/TREATMENT_PROGRAM_EDITOR_DRAFT_SNAPSHOT_BACKFILL.md). Тесты: `instanceEditorDraft.test.ts`, `instanceEditorLoadSettings.test.ts`, `InstanceEditorDraftContext.test.tsx`, `flushInstanceEditorDraft.test.ts`, `instanceEditorPhase2FetchGuard.test.ts`, `InstanceEditorSaveBar.test.tsx` (legacy), `InstanceEditorUnsavedChangesDialog.test.tsx`, `doctor-timeline-text.test.ts`, `TreatmentProgramInstanceDetailClient.phase2.test.tsx`, `TreatmentProgramInstanceDetailClient.phase4.test.tsx`, `TreatmentProgramInstanceDetailClient.phase5.test.tsx`, `TreatmentProgramInstanceDetailClient.phase6.test.tsx`, **`TreatmentProgramInstanceDetailClient.phase7.test.tsx`**, `DoctorProgramInstanceTimelineEventRow.test.tsx`, `InstanceEditorToolbar.test.tsx`, `InstanceEditorAddStageDialog.test.tsx`, `InstanceEditorStageOrderDialog.test.tsx`, `instanceEditorDefaultExpandedStageId.test.ts`, `useInstanceEditorPipelineStageExpansion.test.tsx`, `treatmentProgramLibraryDraftSnapshot.test.ts`, `editorDraftSnapshotDetect.test.ts`, `doctorInstanceRouteErrorStatus.test.ts`; сервер: `instanceEditorBatch.test.ts`, `instanceEditorBatchSchema.test.ts`, `editor-batch/route.test.ts`.
