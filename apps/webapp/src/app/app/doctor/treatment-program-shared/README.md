# treatment-program-shared

Общие примитивы для экранов врача по программам лечения:

- **`treatmentProgramConstructorShellStyles.ts`** — цвета шапок карточек, классы карточек этапа/«общие рекомендации», `tplToolbarTextBtnClass`; алиасы `TPL_*` совпадают с именами в конструкторе шаблона и указывают на те же строки, что `INSTANCE_*`.
- **`TreatmentProgramLibraryPickerToolbar.tsx`** — поиск + регион + тип нагрузки для picker программы/шаблона (**без** «Без региона» / «Без типа» — только на экранах каталогов врача).
- **`InstanceAddLibraryItemDialog.tsx`** — модалка «Элемент из библиотеки» для экрана назначенной программы; добавление через **`addItemCreate`** (in-memory draft), без немедленного POST. Для **упражнений ЛФК и комплексов ЛФК** — toolbar + `useTreatmentProgramLibraryPickerList`. Разворот набора тестов / комплекса ЛФК использует **`expandLines`** в строках каталога (`buildTreatmentProgramLibraryPickers`). Снимки preview: `treatmentProgramLibraryDraftSnapshot.ts`. Тесты: `InstanceAddLibraryItemDialog.test.tsx`, `treatmentProgramLibraryPickerFilters.test.ts`.
- **`treatmentProgramLibraryTypes.ts`** — типы строк каталога для шаблона и инстанса.
- **`TreatmentProgramDndUi.tsx`** — `@dnd-kit` обёртки: DnD pipeline-этапов и элементов этапа (grip на строке/этапе).
- **`treatmentProgramReorderHelpers.ts`** — расчёт `orderedStageIds` / `orderedItemIds` после DnD и chevrons; `planStageItemDndReorder` для PATCH `groupId` + bulk reorder.
- **`InstanceEditorToolbar.tsx`** / **`InstanceEditorAddStageDialog.tsx`** / **`InstanceEditorStageOrderDialog.tsx`** — sticky шапка редактора **назначенной** программы (три зоны: meta | «Комментарии» | actions; `INSTANCE_EDITOR_TOOLBAR_STICKY_CLASS` под `DoctorHeader`). Порядок pipeline-этапов — модалка → draft; inline stage DnD на экране инстанса снят. Pipeline-этапы сворачиваются (`Collapsible`); default expanded — `pickDefaultExpandedPipelineStageId` + `useInstanceEditorPipelineStageExpansion` («+ Группа» раскрывает свёрнутый этап). Legacy `InstanceEditorSaveBar` — `@deprecated`. Тесты: `InstanceEditorToolbar.test.tsx`, `InstanceEditorAddStageDialog.test.tsx`, `InstanceEditorStageOrderDialog.test.tsx`, `instanceEditorDefaultExpandedStageId.test.ts`, `useInstanceEditorPipelineStageExpansion.test.tsx`, `TreatmentProgramInstanceDetailClient.phase4.test.tsx`, `TreatmentProgramInstanceDetailClient.phase5.test.tsx`.
- **`instanceEditorDraft.ts`** / **`InstanceEditorDraftContext.tsx`** — in-memory черновик редактора **назначенной** программы. Сохранение: **`flushInstanceEditorDraft`** → `POST …/editor-batch` (metadata + structural); одно событие `program_changed` (`payload.diff`). Сервер: **`applyInstanceEditorBatch`** в **`runInMutationTransaction`** (PG Drizzle tx / in-memory snapshot); pre-validate до мутаций. **`isFlushableDirty`** — unsaved gate перед status API блокирует только metadata. Тесты: `instanceEditorDraft.test.ts`, `InstanceEditorDraftContext.test.tsx`, `flushInstanceEditorDraft.test.ts`, `instanceEditorPhase2FetchGuard.test.ts`, `InstanceEditorSaveBar.test.tsx` (legacy), `InstanceEditorUnsavedChangesDialog.test.tsx`, `TreatmentProgramInstanceDetailClient.phase2.test.tsx`, `TreatmentProgramInstanceDetailClient.phase4.test.tsx`, `TreatmentProgramInstanceDetailClient.phase5.test.tsx`, `InstanceEditorToolbar.test.tsx`, `InstanceEditorAddStageDialog.test.tsx`, `InstanceEditorStageOrderDialog.test.tsx`, `instanceEditorDefaultExpandedStageId.test.ts`, `useInstanceEditorPipelineStageExpansion.test.tsx`, `treatmentProgramLibraryDraftSnapshot.test.ts`; сервер: `instanceEditorBatch.test.ts`, `instanceEditorBatchSchema.test.ts`, `editor-batch/route.test.ts`.
- Тесты: `treatmentProgramReorderHelpers.test.ts`, `treatmentProgramInstanceItemDnd.test.ts`; RTL шаблона — `TreatmentProgramConstructorClient.reorder.test.tsx`.
- **Конструктор шаблона** (`TreatmentProgramConstructorClient.tsx`): CTA внизу через `DoctorCatalogPersistPublishBar` — у **черновика** «Сохранить черновик» disabled, публикация через «Опубликовать»; у **опубликованного** «Сохранить черновик» (перевод в `draft`) disabled, пока нет несохранённых правок названия/описания (`templateBasicsDirty`; структура этапов пишется в API сразу).
