# diaries

Дневники пациента: симптомы и ЛФК.

Два подмодуля с общей идеей: хранение делегировано портам (реализации в infra/repos — in-memory или PostgreSQL).

- **Симптомы**: создание отслеживания по названию, список отслеживаний, добавление записей (балл 0–10, тип «в моменте»/«за день»), список записей. Вызывается вебаппом (страница дневника симптомов) и API интегратора (бот при создании трекинга и при запросе списка для кнопок 0–10). На стороне UI: журнал записей с клиентской логикой обновления (`symptomDiaryClientEvents`), отдельные клиентские экраны для трекинга и правок (см. `app/patient/diary/symptoms/`, `symptomJournalEditWindow.ts`).
  - На единой странице дневника удалена дублирующая форма «Добавить запись»; добавление выполняется из строки симптома (кнопка `+`) и через quick-add.
  - Статистика симптома в `/api/patient/diary/symptom-stats` агрегируется по дням двумя отдельными сериями: `instant` и `daily` (максимум в рамках типа за UTC-день).
  - График симптома рендерит две линии разного цвета: «В моменте» и «За день».
- **Удаление данных дневников**: в профиле пациента (не в bot-режиме) есть секция destructive-удаления всех дневниковых данных. Поток: подтверждение → PIN → SMS OTP → `POST /api/patient/diary/purge`. Транзакция в `pgDiaryPurge.ts` удаляет `symptom_trackings`, `lfk_complexes` и связанные записи, не затрагивая профиль.
- **ЛФК**: создание комплекса по названию, список комплексов, отметка занятия (сессия), список занятий. Вызывается вебаппом (страница дневника ЛФК, форма «Отметить занятие») и интегратором (бот при создании комплекса и отметке занятия).

Типы и порты описаны в types.ts и ports.ts; сервисы — symptom-service.ts, lfk-service.ts.
