# reminders

Сервисные правила напоминаний в **webapp** (`public.reminder_rules` — источник истины для продуктовых правил), журнал действий (`reminder_journal`), API пациента и integrator M2M. **Фактическая отправка push** и планирование due-occurrence для правил **с ботом** (`integrator_user_id` задан) выполняются в **integrator**; диспетчер читает правила из локальной БД после upsert из webapp и учитывает `platform_users.reminder_muted_until`.

**Web Push без бота** (`integrator_user_id IS NULL`, есть `platform_user_id` и подписка): правило остаётся только в `public.reminder_rules`, integrator **не** планирует occurrence. Планирование и доставка — webapp tick: таблица `webapp_reminder_occurrences`, `planDueReminderOccurrences` (только **будущие** слоты текущего локального дня, без backfill прошлых интервалов) + `runWebPushOnlyReminderTick`, `POST /api/internal/reminders/web-push-only/tick` (Bearer `INTERNAL_JOB_SECRET`, loopback). При смене расписания — `cancelWebPushPendingOccurrences`; tick снимает legacy pending старше 3 мин (`orphaned_past_slot`). Push — `runPlatformUserReminderWebPushNotify` (темы/каналы как у M2M `notify-channels`, только web_push). Idempotency: `UNIQUE(integrator_rule_id, occurrence_key)` на `webapp_reminder_occurrences`. Tick summary: `rulesFound`, `plannedUpserts` / `planned`, `dueClaimed`, `sent`, `skipped`, `skippedNoSubscription`, `skippedNoTopic`, `failed`.

## Наблюдаемость (admin)

- **Краткий срез в «Здоровье системы»:** [`GET /api/admin/system-health`](../../app/api/admin/system-health/route.ts) → поле **`remindersPipeline`** в теле ответа (в т.ч. **`patientReminderM2mIdempotencyKeysActive`** — активные ключи idempotency M2M push/email) и проба **`reminders_pipeline`** в **`meta.probes`** (реализация: [`collectAdminSystemHealthData.ts`](../../app-layer/health/collectAdminSystemHealthData.ts), [`adminReminderPipelineMetrics.ts`](../../app-layer/health/adminReminderPipelineMetrics.ts)); UI — аккордеон «Напоминания» в [`SystemHealthSection.tsx`](../../app/app/settings/SystemHealthSection.tsx).
- **Web Push-only cron tick:** каждый вызов [`POST /api/internal/reminders/web-push-only/tick`](../../app/api/internal/reminders/web-push-only/tick/route.ts) (best-effort) обновляет **`public.operator_job_status`** (`job_key=reminders.web_push_only.tick`); в admin health — **`webPushOnlyReminderTick`** + проба **`web_push_only_reminder_tick`**; блок в аккордеоне «Web Push (PWA)». Пустой успешный tick не пишет `info` в journal.
- **Детальная статистика за окно (`windowHours`):** [`GET /api/admin/reminder-stats`](../../app/api/admin/reminder-stats/route.ts) (admin mode) и **`GET /api/doctor/content-stats`** (doctor/admin без admin mode) — один загрузчик **`loadContentEngagementStats`** ([`loadAdminReminderStats.ts`](../../app-layer/stats/loadAdminReminderStats.ts)); вкладка **`/app/settings?adminTab=reminder-stats`**, компонент [`ReminderStatsSection.tsx`](../../app/app/settings/ReminderStatsSection.tsx). Контракт и поля (в т.ч. **`peopleWithNotifications`**, **`displayTimezone`**; устар. **`reminderRulesEnabledCount`**) — [`api.md`](../../app/api/api.md) (**admin/reminder-stats**, **doctor/content-stats**).
- **M2M fan-out после `dispatchDue`:** `POST /api/integrator/patient-reminders/notify-channels` — реализация [`integratorNotifyChannels.ts`](integratorNotifyChannels.ts) (web push + transactional SMTP; **List-Unsubscribe** при валидном `From`; сглаживание частоты email — `email_send_cooldowns`, ключ **`!reminder_txn_v1`**). Тексты Web Push строит [`pushNotificationCopy.ts`](../web-push/pushNotificationCopy.ts) по `linkedObjectType` / `reminderIntent` / `occurrenceCategory` (разминка, тренировка, custom для legacy-строк в БД, skip `water`/`breathing`/`supplements_medication`).
- **Legacy custom-напоминания:** UI создания/редактирования (`linkedObjectType=custom`) снят с пациентского экрана; существующие правила продолжают dispatch и push copy `custom`. Новые custom через API не создаются.
- **Рассылки врача (Push):** канал `push` в `/app/doctor/broadcasts` → `fanOutBroadcastWebPush` (`intentType: news`, тема `news`). См. `modules/doctor-broadcasts/fanOutBroadcastWebPush.ts`.

## Бот (Telegram / MAX)

- Inline-клавиатура диспатча напоминания: [`reminderInlineKeyboard.ts`](../../../../../integrator/src/kernel/domain/reminders/reminderInlineKeyboard.ts): **Начать разминку/занятие** (web_app), **Через 20 минут | Пропущу**, **Тишина до завтра | Расписание**, **Не напоминать в боте** (`rem_bot_off:<occurrenceId>`), **Установить мобильное приложение** (web_app → `/app/patient`). Отдельной строки **«Выполнить»** на клавиатуре нет (отметку `done` пользователь делает через основной CTA в mini-app / целевой экран); **`rem_done:<occurrenceId>`** и сценарии `telegram.reminder.done` / `max.reminder.done` остаются для **старых сообщений**, где кнопка ещё есть.
- Скип одним нажатием: **`rem_skip`** → script `telegram.reminder.skip.open` / `max.reminder.skip.open` → **`reminders.skip.applyPreset`** с **`reasonCode: none`**; текст ответа — шаблон **`reminder.skip.saved`** (+ fallback в `handlers/reminders.ts`). Выбор причины (`rem_skip_r:*`) остаётся для старых клавиатур.
- **Не напоминать в боте:** **`reminders.messengerTopic.disable.callback`** → signed **`POST /api/integrator/reminders/messenger-topic/disable`** ([`disableReminderMessengerTopic.ts`](disableReminderMessengerTopic.ts)): при наличии **темы** (`reminderOccurrenceTopicCode`, синхронно с integrator [`reminderNotificationTopicCode.ts`](../../../../../integrator/src/kernel/domain/reminders/reminderNotificationTopicCode.ts)) — `topicChannelPrefs.upsert` для пары тема × мессенджер (`user_notification_topic_channels`), ответ пользователю со списком оставшихся каналов; без темы — короткий MVP-текст и те же две deeplink-кнопки профиля/приложения.
- **Integrator** (`reminders.done.callback`, сценарии `telegram.reminder.done` / `max.reminder.done`): после **`ok: true`** от webapp **`POST /api/integrator/reminders/occurrences/done`** — `callback.answer` → **`message.delete`** исходного сообщения (у MAX `message_id` — строка). Если **`firstDoneForOccurrence`**, **`dayFullyDone`**, **`daySentTotal > 0`** — отдельное **`message.send`** с шаблоном **`reminder.dayAllDone`** (`{{done}}`, `{{total}}` в `apps/integrator/src/content/{telegram,max}/user/templates.json`). При ошибке webapp колбэк завершается **failed**, удаления нет.
- **Webapp:** день для агрегатов считается по локальной дате **`reminder_occurrence_history.occurred_at`** в IANA-зоне **`app_display_timezone`** (`getAppDisplayTimeZone`), в той же транзакции что запись `done` в `reminder_journal` — `pgReminderJournal.recordDone`, вызов из `doneOccurrence` в `service.ts`.
- Ответы **200** с полями **`firstDoneForOccurrence`**, **`dayDoneCount`**, **`daySentTotal`**, **`dayFullyDone`**: маршруты [`occurrences/done/route.ts`](../../app/api/integrator/reminders/occurrences/done/route.ts) (integrator) и [`patient/reminders/[id]/done/route.ts`](../../app/api/patient/reminders/[id]/done/route.ts) (пациент).

- `service.ts` — бизнес-логика; порты: `ReminderRulesPort`, опционально `ReminderJournalPort`.
- `resolveReminderIntentForLinkedObject.ts` — при создании object-reminder: `content_section` с `system_parent_code = warmups` → `reminder_intent = warmup`; `rehab_program` → `exercises`; иначе `generic`. Вызывается из `createObjectReminder` (deps `contentSections`).
- `buildReminderDeepLink.ts` — deeplink для payload integrator и web-push open URL. Для `reminder_intent` **`warmup`** / **`exercises`** / **`stretch`** — `/app/patient/go/daily-warmup` и `/app/patient/go/plan-start-lesson`. **Fallback:** `content_section` + slug `warmups` при `generic` (legacy) → тот же go URL. Миграция **`0077_reminder_warmup_intent_backfill`**: backfill `reminder_intent` для правил на разделы кластера warmups.
- `scheduleSlots.ts` — формат `slots_v1`, дефолт реабилитации `DEFAULT_REHAB_DAILY_SLOTS` (09:00 и 19:00, `weekly_mask` 7/7). Дефолты формы «офисный день» для не-rehab — `reminderFormDefaults.ts` (окно 12–18, интервал 180 мин, маска Пн–Пт). При **первом** включении push в PWA — `ensureWarmupsReminderOnFirstPwaPush` (`DEFAULT_WARMUP_PWA_PUSH_ONBOARDING_SLOTS`: 11:00 / 14:00 / 17:00, будни), если правила на разминки ещё нет.
- Репозитории: `infra/repos/pgReminderRules.ts`, `pgReminderJournal.ts`; проекция для integrator GET: `infra/repos/pgReminderProjection.ts` (поля `schedule_data`, `reminder_intent`, `display_*` в ответе списка правил).
